# Rule 012 - 交互节点显性化硬规则

## 规则定义

每个设计方向必须有至少 1 个「显性交互特征」，即使无按钮设计也必须通过隐性显性实现可发现、可理解、可记忆。

## 适用范围

STEP-11 交互设计、STEP-14 审美质量门

## 执行标准

### 强制要求

1. **每个方向必须有 ≥1 个显性交互特征**
   - 类型：显性按钮 / 微凹点 / 纹理变化 / 色彩对比 / 形态暗示 / 动态反馈
   - 即使选择「无按钮」设计，也必须通过微凹点/纹理/形态/动态反馈实现隐性显性

2. **隐性显性设计**
   - 不突兀但可发现
   - 用户能凭直觉知道「这里可以操作」
   - 必须在提示词中精确描述（尺寸、位置、材料、触发方式）

3. **交互反馈必须可感知**
   - 每个高频操作必须有 ≥1 个反馈
   - 反馈类型：触觉微震 / LED 光反馈 / 声音反馈 / 形态反馈 / 温度反馈
   - 反馈必须与操作一一对应

### 禁止行为

- ❌ 「无按钮设计」等模糊描述（必须说明如何无按钮但可交互）
- ❌ 所有交互节点完全隐藏
- ❌ 操作后无反馈
- ❌ 一个反馈对应多个操作

## 违反后果

STEP-11 交互设计不通过，退回重新推导。

## 验证方式

检查清单：
- [ ] 是否有 ≥1 个显性交互特征？
- [ ] 显性交互特征是否「不突兀但可发现」？
- [ ] 用户能否凭直觉知道「这里可以操作」？
- [ ] 每个高频操作是否有 ≥1 个可感知反馈？
- [ ] 反馈是否与操作一一对应？
- [ ] 是否避免了「无按钮」等模糊描述？
