## W.1 关系命题生成器

### 核心模板

```
Step 1: 识别默认品类答案
"这是一个_______________"

Step 2: 问关系问题
- 谁在使用？
- 在什么环境下？
- 需要做什么行为？（用精确动词）
- 要建立什么信任？

Step 3: 重写关系命题
"这不是一个_______________，这是_______________与_______________、_______________、_______________的关系。"
```

### 填写示例

```
产品：咖啡机

Step 1: 品类答案："这是一个咖啡机"

Step 2: 关系问题
- 用户：在家享用咖啡的成年人
- 环境：厨房，每日早晨
- 行为：研磨、冲泡、享用
- 信任：可靠的早晨仪式

Step 3: 关系命题
"这不是一个咖啡机；这是一个成年人与研磨、冲泡、享用咖啡的早晨仪式，在厨房环境中，建立的是可靠陪伴的信任。"
```

## W.4 CPSO速写模板

### 10分钟版

```
产品：_______________
日期：_______________

【Context】情境：_______________
【Problem】核心矛盾：_______________
【Solution】决策逻辑：_______________
【Outcome】判断验证：_______________
```

### 30分钟版

```
# 案例：[产品名称]

## Context (背景)
情境要素：时间压力/信息完整度/利益相关方/可用资源
关键理解：这不是...而是...

## Problem (问题)
核心矛盾：表层问题 + 深层矛盾
判断题：真正挑战是什么？

## Solution (方案)
决策逻辑：1/2/3
备选方案：❌方案A → 因为... / ✅最终方案 → 因为...

## Outcome (结果)
短期/长期/意外结果
判断验证：✓假设 + ✓边界
```

## W.5 判断决策树

```
Q1: 能说清楚关系吗？ → No则重写关系
Q2: 材料有职责吗？ → No则检查材料
Q3: 触觉有确认吗？ → No则增加节点
Q4: 十年能维护吗？ → No则重新考虑
Q5: 克制是力量吗？ → No则减少噪音
→ ✓通过
```

### 使用说明

```
【何时使用】
- 评估一个设计方案时
- 发现设计有问题时
- 需要快速检查时

【如何执行】
1. 从Q1开始，逐个问题检查
2. 如果答案No，在该步骤修正
3. 修正后再继续下一步
4. 全部通过则方案可行
```
