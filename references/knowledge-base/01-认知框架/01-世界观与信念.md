## 1.2 设计哲学 ⭐

### 核心命题 [S]

```
设计的第一性原理：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
先重写人与产品的关系，
再让材料、结构、触觉、语言、维护和协作共同证明这个关系。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 四大约束 ⭐ 新增自艾维资料全集

```
约束 1: 产品不是外观题，而是关系题
─────────────────────────────────────────────
灯 → 人与光、携带、熄灭、海洋环境的关系
车 → 人与 Ferrari EV 架构、驾驶仪式的关系
唱盘 → 人与经典机械仪式的关系
Seal → 制度、价值观和授予界面的关系
AI → 人与计算界面的关系

约束 2: 熟悉感不能靠怀旧复制
─────────────────────────────────────────────
可引用：archetype、ritual、instrument、ceremonial
必须附：not nostalgic replica / not retro prop

约束 3: 触觉不是装饰
─────────────────────────────────────────────
旋钮/铰链/按钮/钥匙/拨杆/织带/压印/纸张
必须承担行为，不只是装饰

约束 4: 克制不是少做
─────────────────────────────────────────────
大量思考、工程、协作 → 安静的结果
不是用造型语言炫耀
```

## 1.3 思维模型

### 模型 1: 关系命题思维

```
问题分析路径：
┌──────────────────┐
│   收到设计 brief  │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 问：这是什么的    │  ← 不是"这是台灯"
│  关系命题？       │    是"人与光/携带/熄灭/环境"
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 问：默认品类答案  │  ← 不是台灯造型
│  是什么？        │    是人与光的仪式
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 问：如何重新组织  │  ← 用 LED + Fresnel + 织带
│  这个关系？       │    而非台灯外壳
└──────────────────┘
```

### 模型 3: 触觉节点思维

```

│           高频触觉节点清单                   │

│                                            │

│  □ 钥匙 (key)     - 开锁、启动              │
│  □ 铭牌 (badge)   - 身份、信任             │
│  □ 旋盖 (cap)     - 密封、开启             │
│                                            │
│  每个节点必须回答：                         │
│  ├── 这个触觉确认什么行为？                 │
│  ├── 阻尼/反馈是什么？                     │
│  ├── 位置/形状如何帮助识别？               │
│  └── 声音/触感传达什么状态？               │
└────────────────────────────────────────────┘
```

### 设计失败检测规则 ⭐ 新增自设计哲学与方法论.md

#### 通用失败

| 失败模式 | 判定标准 | 修正指令 |
|----------|----------|----------|
| 风格词替代结构 | 只有minimal/premium/sleek/Apple-like | 重写为比例+分层+控制节点 |
| 材料无职责 | 金属/玻璃/木/纸只作为装饰 | 为每种材料指定功能职责 |
| 交互消失 | 没有旋钮/按钮/铰链等 | 增加触觉节点 |
| 过度无缝 | 螺丝/铰链/接缝被抹掉 | 增加honest fastening |

#### 高风险误读

| 误读类型 | 为什么错误 | 如何避免 |
|----------|------------|----------|
| Apple-like | LoveFrom可以做复杂Seal/航海灯/唱盘铰链 | 用关系/材料职责/触觉动作替代风格词 |
| 复古怀旧 | 熟悉感来自动作和秩序，不是复制风格 | 引用archetype但附not nostalgic replica |
| 极简=减法 | 克制是准确表达，不是减少元素 | 检查复杂元素是否被秩序组织 |
| 外包=主导 | LoveFrom是合作方，设计主权在客户 | 写合作方擅长什么而非Ive会怎么画 |

## 1.5 隐性知识萃取

### 艾维"知道但不说"的事 [B]

```
基于案例分析和行为观察萃取的隐性知识：

隐性知识 1: 如何判断一个方案"太像 Apple"
─────────────────────────────────────────────────────
表现：
- 大量使用白色
- 追求圆角
- 追求无缝
- 追求极薄

背后逻辑：[A]
- Apple 是为了统一生态系统
- 模仿 Apple 外观是误解 Apple 方法
- Apple 方法是"减少选择"而非"统一风格"

隐性知识 2: 为什么讨厌"smart"产品 [B]
─────────────────────────────────────────────────────
表现：
- 从不称产品为"smart"
- 强调"traditional interface"和"physical control"

背后逻辑：[B]
- "smart"暗示产品在做决定
- 艾维认为产品应该响应人的决定
- 不是智能，是诚实

隐性知识 3: 为什么坚持实体零售店 [B]
─────────────────────────────────────────────────────
表现：
- 参与 Apple Store 设计
- 强调"人与产品的物理接触"

背后逻辑：[A]
- 触觉是建立信任的媒介
- 在店里拿起产品的那一刻是关键的信任时刻
- 零售空间是产品的延伸

隐性知识 4: 为什么关注"消失" [S]
─────────────────────────────────────────────────────
表现：
- AirPods 戴用后几乎不可见
- Apple Park 让建筑"消失"
- 界面应该让位

背后逻辑：[S][A]
- 产品不应该抢走人对任务的注意力
- 最好的设计是让人忘记产品的存在
- 消失是给注意力让路

隐性知识 5: 为什么重视"手工"和"匠人" [B]
─────────────────────────────────────────────────────
表现：
- 父亲是银匠
- 经常提到"making"
- 参与手工制作

背后逻辑：[A]
- 手工意味着人的判断
- 机器量产意味着去人格化
- 产品应该承载制作者的判断
```
