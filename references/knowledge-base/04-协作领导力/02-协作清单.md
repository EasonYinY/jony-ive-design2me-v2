# 04-补充. 协作清单

> **版本**：1.0
> **日期**：2026-06-22
> **定位**：协作前中后的完整检查清单与操作指南
> **目标**：让协作过程可追踪、可评估、可优化
> **来源**：艾维协作经验 × LoveFrom模式

---

## 0. 协作清单概述

### 0.1 为什么需要协作清单

```
┌─────────────────────────────────────────────────────────────────────┐
│                    协作的三个关键时刻                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【协作前】                                                       │
│  充分了解 → 正确进入                                            │
│  → 不了解就进入 = 盲目协作                                      │
│  → 了解后才进入 = 有准备的协作                                  │
│                                                                     │
│  【协作中】                                                       │
│  持续验证 → 保持方向                                            │
│  → 只做不看 = 执行偏离                                          │
│  → 边做边看 = 及时调整                                         │
│                                                                     │
│  【协作后】                                                       │
│  能力转移 → 确保持续                                            │
│  → 做完就走 = 外包思维                                          │
│  → 帮对方成长 = 协作思维                                        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 0.2 清单架构

```
协作清单
│
├── 协作前清单（进入前必查）
│   ├── 合作方身份评估
│   ├── 协作目标定义
│   ├── 进入深度选择
│   └── 风险评估
│
├── 协作中清单（执行中检查）
│   ├── 方向检查
│   ├── 关系检查
│   └── 能力检查
│
└── 协作后清单（退出前验证）
    ├── 能力转移验证
    ├── 持续性评估
    └── 经验萃取
```

### 4.1 争议处理

```
┌────────────────────────────────────────────────────────────────────┐
│                    特殊情境 · 争议处理                            │
├────────────────────────────────────────────────────────────────────┤
│                                                                    │
│  【争议类型识别】                                                 │
│  □ 是判断争议还是偏好争议？                                       │
│  □ 是设计问题还是商业问题？                                       │
│  □ 是短期分歧还是长期矛盾？                                       │
│                                                                    │
│  【处理原则】                                                     │
│  ├── 用作品说话，而非语言辩论                                    │
│  ├── 争议的是"什么是正确"，不是"谁对谁错"                      │
│  ├── 有共同的判断标准，争议促进正确                              │
│  └── 保持专业，不涉及个人                                        │
│                                                                    │
│  【处理流程】                                                     │
│  Step 1: 理解争议的本质                                          │
│  Step 2: 用原型/模型展示，让作品说话                              │
│  Step 3: 找共同目标，而非对抗                                    │
│  Step 4: 如果无法达成共识 → 找更高层判断                         │
│  Step 5: 记录决策和原因，作为未来参考                            │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```
