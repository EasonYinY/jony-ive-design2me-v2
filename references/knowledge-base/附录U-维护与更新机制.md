```

- 架构重组
```

## 4. 贡献机制（详细流程）

### 4.1 贡献类型

```
┌─────────────────────────────────────────────────────────────────────┐
│                    贡献类型说明                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  类型A：新案例贡献                                               │
│  ├── 说明：提交新的产品案例完整CPSO分析                          │
│  ├── 要求：完整四要素+判断逻辑+方法论验证                       │
│  └── 评审：需要方法论评审                                       │
│                                                                     │
│  类型B：内容更新                                                 │
│  ├── 说明：更新现有文档内容                                      │
│  ├── 要求：保持格式和置信度标注一致                              │
│  └── 评审：需要变更说明                                         │
│                                                                     │
│  类型C：新增章节                                                 │
│  ├── 说明：新增独立文档或章节                                    │
│  ├── 要求：符合框架结构，有完整索引                             │
│  └── 评审：需要架构评审                                         │
│                                                                     │
│  类型D：误用反馈                                                 │
│  ├── 说明：报告误用案例和修正过程                                │
│  ├── 要求：详细描述误用模式和修正方法                           │
│  └── 评审：需要实用性评审                                       │
│                                                                     │
│  类型E：跨领域迁移                                               │
│  ├── 说明：将方法论迁移到新领域                                 │
│  ├── 要求：保持方法论核心，更新应用场景                         │
│  └── 评审：需要应用性评审                                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 4.2 贡献流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                    贡献流程                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Step 1: 准备 (Prepare)                                          │
│  ├── 阅读贡献指南                                                │
│  ├── 确认贡献类型                                                │
│  └── 准备贡献内容                                                │
│                                                                     │
│  Step 2: 编写 (Draft)                                           │
│  ├── 按照模板编写                                                │
│  ├── 标注来源和置信度                                           │
│  └── 添加索引                                                    │
│                                                                     │
│  Step 3: 提交 (Submit)                                          │
│  ├── 创建贡献文档                                                │
│  ├── 填写贡献说明表                                             │
│  └── 提交评审                                                   │
│                                                                     │
│  Step 4: 评审 (Review)                                          │
│  ├── 准确性检查                                                 │
│  ├── 完整性检查                                                 │
│  ├── 格式检查                                                   │
│  └── 实用性评估                                                 │
│                                                                     │
│  Step 5: 合并 (Merge)                                           │
│  ├── 评审通过                                                   │
│  ├── 版本更新                                                   │
│  └── 合并发布                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 5. 评审标准（详细检查项）

### 5.1 准确性评审

```
┌─────────────────────────────────────────────────────────────────────┐
│                    准确性评审检查项                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【事实正确性】                                                  │
│  □ 案例事实有可靠来源                                           │
│  □ 引语来自艾维原话或可靠资料                                   │
│  □ 日期、品牌、产品信息准确                                     │
│                                                                     │
│  【推断合理性】                                                  │
│  □ [A]级推断有方法论支撑                                       │
│  □ [B]级假设已明确标注                                          │
│  □ 推断逻辑清晰可追溯                                           │
│                                                                     │
│  【一致性检查】                                                  │
│  □ 与现有内容不冲突                                             │
│  □ 与16法核心原则一致                                           │
│  □ 与艾维设计哲学一致                                           │
│                                                                     │
│  【客观性检查】                                                  │
│  □ 避免主观臆断                                                 │
│  □ 区分事实和推断                                               │
│  □ 避免过度解读                                                 │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
