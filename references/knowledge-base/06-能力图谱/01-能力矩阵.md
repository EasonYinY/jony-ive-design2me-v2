## 2. 能力详解

### 2.1 设计判断力

```
定义：准确判断设计方案是否"正确"的能力 [S]

核心表现：
├─ 能从"关系命题"而非"外观"角度判断
├─ 能识别"Apple-like"误用
├─ 能判断材料是否有职责
├─ 能判断触觉是否承担行为
└─ 能判断克制是否到位

评估标准：
□ 方案判断准确率
□ 判断一致性（不同时间/情绪下判断相同）
□ 判断可解释性（能说出为什么对/错）
□ 判断前瞻性（十年后判断仍正确）
```

**深度理解** [A]:
```
表面：设计判断力 = 看方案的能力

深度理解：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
什么是"正确"的判断？
→ 不是"好看"
→ 是"关系命题是否正确"
→ 是"材料是否有职责"
→ 是"触觉是否承担行为"

判断标准从哪来？
→ 从16条方法论
→ 从案例库
→ 从价值观

怎么培养？
→ 大量案例分析
→ 建立判断标准
→ 验证判断准确性
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 2.2 材料工艺

```
定义：对材料性能、感官、职责的深度理解 [S]

核心表现：
├─ 能说清每个材料的"职责"
├─ 能判断材料是否"诚实"
├─ 能平衡功能/情绪/时间
├─ 能创新使用传统材料
└─ 能理解材料的限制

评估标准：
□ 材料知识广度
□ 材料职责识别准确度
□ 材料组合创新能力
□ 材料局限性认知
```

**深度理解** [A]:
```
表面：材料工艺 = 认识材料

深度理解：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
什么是"材料职责"？
→ 功能职责：这个材料做什么
→ 情绪职责：这个材料传达什么
→ 时间职责：这个材料十年后变成什么

为什么要写"for..."？
→ 强迫思考
→ 不是"高级材料"
→ 是"精确描述"

常见错误：
→ 写"premium metal"
→ 不写"for structural integrity"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 2.3 触觉交互

```
定义：设计和评估触觉体验的能力 [S]

核心表现：
├─ 能设计触觉节点承担行为
├─ 能判断触觉是否"诚实"
├─ 能平衡视觉和触觉
├─ 能设计触觉反馈
└─ 能理解触觉的文化差异

评估标准：
□ 触觉节点识别能力
□ 行为-触觉映射准确性
□ 触觉反馈设计能力
□ 触觉-视觉平衡能力
```

**深度理解** [A]:
```
表面：触觉交互 = 注意触感

深度理解：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
为什么触觉是"信任的媒介"？
→ 手是最诚实的感官
→ 眼睛可以被欺骗
→ 手会告诉你真相

什么是"触觉节点"？
→ 用户的手会反复触摸的地方
→ 高频交互点
→ 状态确认点

触觉节点必须回答：
→ 这个触觉确认什么行为？
→ 反馈是什么？（声音/阻尼/位置）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 2.4 协作领导力

```
定义：与他人合作并领导团队的能力 [A]

核心表现：
├─ 能进入合作方系统
├─ 能建立长期协作关系
├─ 能用作品而非语言说服
├─ 能培养合作方能力
└─ 能在争议中保持专业

评估标准：
□ 协作关系深度
□ 合作方成长度
□ 团队满意度
□ 争议处理能力
```

**深度理解** [A]:
```
表面：协作领导力 = 沟通能力

深度理解：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
为什么"协作是设计对象"？
→ 协作方式影响设计结果
→ 驻场 vs 外包 = 不同结果
→ 帮对方成长 vs 替对方做 = 不同结果

什么是"用作品说话"？
→ 语言可以被辩论
→ 作品可以直接体验
→ 体验比辩论更有说服力

如何"帮对方成长"？
→ 不是给方案
→ 是教方法
→ 让对方在LoveFrom离开后更强
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

├─ 能统一视觉语言
├─ 能平衡一致性和创新性
├─ 能通过视觉传达品牌

**深度理解** [A]:
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

什么是"关系清楚"？
→ 不是元素少

→ 秩序 = 克制

如何平衡一致性和创新？
→ 一致性 = 品牌关系
→ 创新 = 解决方案
→ 两者不冲突
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

## 3.5 项目级能力补强（2024-2026 新增） [A]

> **来源**：Moncler × LoveFrom（2024-09）+ William Stout × LoveFrom（2025-05）+ Christie's rostrum（2026-03）
> **等级**：A（CEO/设计师三方直接引语 + 多源印证）

### 3.5.1 小构件极限设计能力（Moncler duo button）[A]

```
┌─────────────────────────────────────────────────────────────────────┐
│           Moncler duo button · 5 年研究 + 1 个构件                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  形 态：5 个双面磁吸 disc + 中央 dowel 定位销                      │
│  位置：前 / 后 / 袖各分布                                            │
│  闭合：内层和外层通过 dowel 自动吸附、严丝合缝                      │
│  隐藏：单穿时 disc 平贴布面（视觉上"消失"）                        │
│  双向品牌：内侧 LoveFrom logo + 外侧 Moncler logo                  │
│                                                                     │
│  极 限 难 度：                                                     │
│  → 5 个 disc × 2 面 = 10 个磁吸面                                   │
│  → 中央 dowel 直径公差 < 0.1mm                                       │
│  → 严丝合缝 = 视觉无可见间隙                                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**艾维原话** [A]（Fast Company 2024-09-06）：
> "I became obsessed with buttons. It's an idea that is hundreds and hundreds of years old, and maybe it's perfect, but as a designer, you want to challenge that assumption."

**Fast Company 转引** [A]：
> "There wasn't some arrogant ambition around disruption [of buttons]. It was a very gentle, humble exploration."

**这个按钮体现了什么能力**：

| 能力维度 | duo button 的具体体现 | 对应方法 |
|----------|----------------|----------|
| 设计判断力 | 不接受"按钮 = 现有方案" | 方法 1（语言先行） |
| 材料工艺 | 5 disc + dowel 精确定位 | 方法 5（材料诚实） |
| 触觉交互 | 单穿平贴 + 加壳吸附，两种触觉并存 | 方法 6（触觉优先） |
| 视觉传达 | 严丝合缝 = 视觉"无" | 方法 9（克制是力量） |
| 长期主义 | 双向 logo = 5 年协作的承诺 | 方法 7（维护是生命） |

**反例/误读风险**：
- ❌ 不要把"duo button"模仿成"磁吸服装"——核心是 5 disc + 中央 dowel 的精确定位，不是磁吸本身
- ❌ 不要把"5 年研究 1 个 button"读成"过度设计"——5 年是协作时长，不是研发时长

### 3.5.2 自研字体能力（William Stout LF Washington）[A]

```
┌─────────────────────────────────────────────────────────────────────┐
│              LF Washington · 字体作为设计对象                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  源 字 体：Russell Bean 1970 设计的 Washington                      │
│  原 用 途：Stout Books 搪瓷招牌                                    │
│  重 设 计：Antonio Cavedoni（曾在 Apple 开发 San Francisco 字体）   │
│  字 体 形 态：几何 sans-serif，基于 Washington lettering            │
│  跨 平 台：印刷 + 数字                                              │
│                                                                     │
│  字体 ≠ 字体：                                                       │
│  → 字体是品牌语调的基础设施                                         │
│  → 字体是公共图形系统的"基石构件"                                  │
│  → 字体是"in-house" 能力信号（不是商用授权）                       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**Chris Wilson（LoveFrom 字体/视觉主创）[A]**：
> "我们很荣幸能帮助诠释这家书店光明的未来，创造一种视觉语言，反映对未来 50 年的乐观，同时尊重之前的 50 年。"

**这个字体项目对应什么能力**：

| 能力维度 | LF Washington 的具体体现 | 对应方法 |
|----------|-------------------|----------|
| 视觉传达 | 字体 = 品牌语调基础设施 | 方法 14（公共图形系统） |
| 协作领导力 | 跨组织字体协作（LoveFrom + Stout + EIIC） | 方法 10（协作是设计对象） |
| 长期主义 | 50 年档案 → 50 年未来 | 方法 7（维护是生命） |
| 熟悉但非怀旧 | 1970 Washington → 2025 LF Washington | 方法 3（熟悉但非怀旧） |

**对应 16 法的填充价值**：
- 方法 1（语言先行）："Stout Books" → "Est. 1974" 框定 50 年关系
- 方法 4（边界即尊重）："Stout 50 年档案"边界 = 尊重历史但不模仿
- 方法 8（小构件大命题）：LF Washington 字体 = 整套识别系统的基石
- 方法 14（公共图形系统）：字体 + logo + 插图 + 配色 + 标牌 = 完整可规范化系统
