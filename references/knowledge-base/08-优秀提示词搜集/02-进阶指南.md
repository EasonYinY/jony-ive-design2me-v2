## 1. 关系命题编写进阶

### 1.1 品类到关系的思维转换

```
品类思维陷阱：
"设计一个台灯" → 触发脑中台灯原型 → 圆锥形灯罩+底座+开关 → 形态被固化

关系思维：
"这不是一个台灯，这是人与光的关系"
→ 光需要做什么？
→ 人在什么场景下需要光？
→ 光如何建立信任？
→ 形态从关系推导
```

### 1.2 关系命题四要素精讲

#### 用户（Person）定义三层

```
第一层-人口统计："成年人"、"儿童"、"老人" → 太宽泛
第二层-使用场景："在海上航行的人"、"在城市通勤的人" → 更具体
第三层-关系角色："需要可靠光源陪伴航行的人" → 最佳
```

#### 行为（Action）精确度

```
层级1-功能描述："用户可以开关灯" → 无法指导形态
层级2-通用动词："用户调节灯光" → 不够精确
层级3-精确动词："用户extinguish光源" → 最佳

精确动词示例：extinguish暗示仪式感，需要旋钮/拨杆而非触屏
```

#### 环境（Environment）转化

```
海洋环境 → 需要盐雾耐腐蚀 → 需要防水设计 → 需要耐高湿度
城市通勤 → 需要轻便 → 需要便携性 → 需要快速使用
```

#### 信任（Trust）类型

```
功能性信任：可维修性、耐用性、可靠性
情感性信任：诚实设计、不欺骗、符合预期
时间性信任：可老化性、可升级性、可持续性
身份性信任：品牌一致性、克制设计、长期美学
```

### 1.3 关系命题编写流程

```
Step 1: 接受品类，立即超越
"好的，设计一个[品类]"
→ 立即追问："但这不是普通的[品类]，这是什么关系？"

Step 2: 定义用户关系
用户是谁？用户与这个品类现在是什么关系？这个关系中缺失什么？

Step 3: 定义行为
用户需要做什么行为？高频的是什么？用精确动词替代通用动词

Step 4: 定义环境和约束
使用环境是什么？环境带来了什么约束？这些约束如何转化为设计要求？

Step 5: 定义信任
这个关系需要建立什么信任？功能性/情感性/时间性/身份性？信任的证据是什么？

Step 6: 整合为命题
"这不是一个[品类]；这是[用户]与[行为]、[环境]、[信任]的关系。"
```

## 2. 材料职责编写进阶

### 2.1 材料选择决策树

```
Q1: 这个位置需要什么功能？
- 光学功能 → 透光率/折射率/散射性
- 结构功能 → 强度/刚度/韧性
- 防护功能 → 防水/防尘/耐腐蚀
- 热学功能 → 散热/保温/耐温
- 声学功能 → 隔音/共鸣/吸收

Q2: 这个位置传达什么情绪？
- 信任感 → 金属/玻璃/石材
- 温暖感 → 木材/皮革/织物
- 轻盈感 → 塑料/铝合金/碳纤维
- 专业感 → 金属/哑光表面/精确加工
- 自然感 → 木材/竹材/石材/羊毛

Q3: 这个位置如何老化？
- 好看的老化 → 不锈钢/铜/真皮革
- 功能稳定 → 阳极氧化铝/粉末涂层
- 需要更换 → 塑料/织物/橡胶
- 避免老化 → 玻璃/陶瓷/不锈钢
```

### 2.2 材料组合逻辑

```
组合原则1-功能互补：不锈钢护架+Fresnel透镜 → 结构+光学
组合原则2-情绪协调：实木底座+金属框架+织物绳 → 温暖+精确+亲和
组合原则3-老化兼容：铜金属+真皮表带 → 都会形成好看的包浆
组合原则4-成本合理：主体用不锈钢，附件用铝合金 → 按需分配
```

### 2.3 材料职责写作模板

```
完整模板：[材料名称]
for [功能职责]
for [情绪品质]
for [时间老化]

范例-便携音箱：
anodized aluminum enclosure
for structural integrity and corrosion resistance (功能)
for precise machined aesthetic (情绪)
for lasting appearance without rust (时间)

woven acoustic fabric grille
for sound transmission without distortion (功能)
for warm approachable texture (情绪)
for cleanability and stain resistance (时间)

silicone control ring
for grip and tactile feedback (功能)
for playful accessible interaction (情绪)
for replacement when worn (时间)
```

## 3. 触觉节点编写进阶

### 3.1 触觉节点选择决策流程

```
Step 1: 识别所有交互点
用户会触碰这个产品的哪些位置？需要确认什么？

Step 2: 分类交互频率
高频（每天多次）：开关操作、音量调节、状态确认
中频（每周数次）：功能切换、模式选择、设备连接
低频（偶尔）：初始设置、深度清洁、维修维护

Step 3: 匹配触觉类型
高频 → 需要快速确认 → 旋钮/拨杆/大面积按钮
中频 → 需要清晰反馈 → 按钮/拨杆
低频 → 需要精确控制 → 隐藏按钮/设置界面

Step 4: 定义反馈形式
位置反馈：手指能轻松找到
状态反馈：知道当前处于什么状态
动作反馈：知道操作已生效
```

### 3.2 触觉节点表达模板

```
基本描述：[节点类型] positioned at [位置]
confirms [行为]
with [反馈形式]

完整描述：Precision-machined knob positioned at top center
confirms brightness and color temperature
with rotational detents providing tactile increments
at 12 discrete positions
with audible click at each position
allowing one-handed operation

反馈词汇：
段落感：detent/click/clicky
连续感：smooth/silk/buttery
阻尼感：damped/resistance
按压感：tactile/push/click
吸附感：magnetic/snapping
声感：click/clunk/thunk
```

### 3.3 触觉设计五大误区

```
误区1-触觉太多：识别最重要的3个触觉节点，其他简化
误区2-触觉不一致：建立统一的触觉语言，类似功能用类似触觉
误区3-忽略盲操作：位置是否符合人体工学，形状/大小是否有区分
误区4-反馈过弱：增加声音反馈/阻尼强度/位置档位感
误区5-高频用触屏：区分触屏和物理控制的使用场景
```

## 4. 迭代与优化技巧

### 4.1 AI反馈诊断流程

```
诊断1-关系命题被忽略：
症状：AI输出了传统品类形态，没有体现关系命题
检查：关系命题是否放在Prompt开头？是否足够具体？
修正：在关系命题后加"This is the ONLY relationship to consider."

诊断2-材料用禁止词：
症状：输出中包含premium/luxury/crystal
检查：Prompt中是否用了这些词？是否明确禁止？
修正：在禁止词列表后加"Any use of these words will be rejected."

诊断3-触觉缺失：
症状：没有触觉节点描述，触觉描述太表面
检查：Prompt中是否明确要求触觉节点？是否给了数量要求？
修正：明确要求"Include at least 3 tactile nodes."
```

### 4.2 渐进式迭代

```
第一轮-基础骨架：只给关系命题和核心约束，观察AI解读
第二轮-补充细节：保留好的，去除误解的，补充缺失的，明确禁止项
第三轮-强化重点：对重要点加重描述，添加验证性问题，给出成功标准
第四轮-精修语言：优化用词，确保清晰，检查歧义，最终输出
```

### 4.3 失败Prompt修复

```
失败Prompt：
"设计一个台灯，极简风格，白色金属材质"
问题：极简风格=品类思维，白色金属=禁止词不明确，没有关系命题

修复Prompt：
"Design a portable light source for desk use.

This is not a desk lamp; this is a relationship between a person and focused illumination, extended comfort, and reliable presence.

Prohibited words: minimalist, simple, clean, white metal, premium, luxury, sleek

Material requirements:
Each material must answer: for what function? for what emotional quality? for how it ages?

Include at least 3 tactile nodes with specific feedback descriptions.

Verify: can the relationship be explained in one sentence?"
```

## 5. 质量检查清单

```
发送前检查：

关系命题：
□ 是否具体？能用一句话说清楚？
□ 是否避免了品类思维？有"not...but..."结构？

材料职责：
□ 材料是否有"for"连接？
□ 材料是否有三个维度（功能/情绪/时间）？

触觉节点：
□ 触觉节点是否≥3个？
□ 每个触觉节点是否有反馈描述？

约束规则：
□ 禁止词是否完整？
□ 克制边界是否定义？
□ 时间检查是否完整？

验证：
□ 是否有验证问题？
□ 是否有成功标准？
```
