```

❌ 错误 Prompt
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
设计一个极简风格的台灯，
白色，圆角，金属材质。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✓ 正确 Prompt
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

这不是台灯，是移动中的仪式光源。

关系要素：
- 行为：提着走、悬挂、熄灭、调校
- 环境：海洋/户外/夜间
- 信任：陪伴、可靠、可维修
```

## 2. 生成检查清单

### 2.1 关系命题检查

```
在输出设计后，必须验证：

□ 能用一句话说清楚这是什么关系吗？
□ 形态是否服务于关系？
□ 关系命题中的每个要素是否都有对应的设计决策？

关系命题模板：
"这不是一个[默认品类]，
这是[用户]与[行为]、[环境]、[信任]的关系。"

如果说不清楚 → 重写关系命题
```

### 2.2 材料职责检查

```
每个材料必须回答：

□ 能说出这个材料在这里做什么功能吗？
□ 能说出这个材料传达什么情绪吗？
□ 能说出这个材料十年后会变成什么样吗？
□ 是否使用了禁止词？

材料职责模板：
[材料] for [具体功能]
[材料] for [情绪品质]
[材料] for [时间老化]

如果无法回答 → 材料选择有问题
```

### 2.3 触觉节点检查

```
每个产品必须有3个以上触觉确认节点。

每个节点必须回答：

Q1: 这个触觉确认什么行为？
Q2: 反馈是什么（声音/阻尼/位置）？
Q3: 位置/形状如何帮助识别？
Q4: 这个触觉节点在高频还是低频交互中？

触觉节点类型：
□ 旋钮 (dial)     → 控制行为确认
□ 按钮 (button)   → 功能选择确认
□ 拨杆 (switch)   → 状态切换确认
□ 铰链 (hinge)    → 开合确认
□ 织带 (strap)    → 携带确认
□ 压印 (emboss)   → 品牌确认

如果少于3个 → 增加触觉节点
```

### 2.4 时间友好检查

```
问：这个产品十年后还能被维护吗？

□ 产品能被打开维修吗？
□ 螺丝是否可见且可操作？
□ 关键部件是否可单独更换？
□ 织带/皮革等易耗品是否可更换？
□ 材料会老化得好看吗？

如果不能 → 重新考虑可维护性
```

## 3. 迭代修正指南

### 3.1 "Apple-like" 修正流程

```
当AI输出出现"Apple-like"误用时：

Step 1: 识别信号
├── 大量使用白色
├── 追求圆角
├── 追求无缝
├── 追求极薄
└── 使用"Apple风格"作为设计理由

Step 2: 重置关系
- 忽略当前形态
- 重新问：这是什么关系？

Step 3: 重新推导
- 关系 → 行为 → 材料 → 形态
- 不是：形态 → 材料 → 关系

Step 4: 验证
- 如果去掉Apple参照，这个设计还成立吗？
- 如果答案是"不成立"，则是误用
```

### 3.2 材料描述修正流程

```
当AI输出使用禁止词时：

Step 1: 识别禁止词
- premium / luxury / crystal / elegant / high-end / sleek

Step 2: 追问职责
- 这个材料在这里做什么？
- 用"for what"改写

Step 3: 改写模板
- [原] premium glass
- [改] precision-ground glass for even light diffusion

Step 4: 验证
- 能说出"for what"吗？
- 如果不能 → 选择有问题
```

### 3.3 触觉缺失修正流程

```
当AI输出缺乏触觉节点时：

Step 1: 识别高频交互
- 用户会反复做什么？
- 用户会用什么确认状态？

Step 2: 添加触觉节点
- 选择节点类型（旋钮/按钮/拨杆/织带等）
- 定义行为确认需求
- 设计反馈形式

Step 3: 验证
- 用户的手能在3个以上位置确认状态吗？
- 如果不能 → 增加节点
```

```
设计一个现代极简风格的台灯，
白色，圆形，金属材质，
营造高级感。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✓ 好 Prompt
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
设计一个人与光、桌面氛围、情感连接关系的灯具。

- 环境：桌面、夜间、私密空间
```

## 5. AI生成专用提示词模板

### 5.1 产品设计模板

```
【艾维风格 · 产品设计 Prompt】

请设计一个[产品类型]，但不是[默认品类答案]。

这是一个[用户]与[行为]、[环境]、[信任]的关系。

【关系定义】
- 用户是谁？
- 核心行为是什么？
- 使用环境是什么？
- 要建立什么信任？

【材料要求】
每个材料必须回答"for what"：
1. [材料] for [功能] + [情绪] + [时间]
2. [材料] for [功能] + [情绪] + [时间]
3. [材料] for [功能] + [情绪] + [时间]

禁止词：premium / luxury / crystal / elegant / sleek / seamless / minimal

【触觉节点】
至少3个触觉确认点：
1. [节点类型] 确认 [行为]，反馈 [形式]
2. [节点类型] 确认 [行为]，反馈 [形式]
3. [节点类型] 确认 [行为]，反馈 [形式]

【时间检查】
- 可维修吗？
- 可更换易耗品吗？
- 会老化得好看吗？
```

### 5.2 视觉语料生成模板

```
【艾维风格 · 图像 Prompt】

[产品类型], photorealistic, soft studio lighting,

the relationship between human and [核心功能],

not [默认品类答案], but [重新定义的关系],

archetype, ritual, but not nostalgic replica,

visible craftsmanship, not hidden manufacture,

[材料] for [功能], [材料] for [情绪], [材料] for [时间老化],

tactile detail showing [触觉节点类型],
the [节点形式] provides [功能] with [反馈形式],
positioned for [使用方式],

the form follows function with precision.
```

## 6. 常见问题与修正

| 问题 | 表现 | 修正方向 |
|------|------|----------|
| Apple-like | 白色/圆角/无缝 | 重写关系命题 |
| 材料"高级" | premium/luxury描述 | 改写为"for功能" |
| 触觉缺失 | 没有交互确认设计 | 增加3+触觉节点 |
| 过于堆砌 | 元素多但关系乱 | 用秩序收束 |
| 无边界 | 没有限制条件 | 添加边界清单 |
| 消失误用 | 只写invisible | 定义让位什么 |
