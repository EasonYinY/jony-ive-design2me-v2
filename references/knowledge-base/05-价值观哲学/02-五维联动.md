## 3. 迭代时机判断

### 3.1 坚持 vs 放弃

```
艾维的迭代哲学：

┌─────────────────────────────────────────────────────────────────────┐
│                    迭代时机判断                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  【什么时候坚持？】                                                │
│  □ 关系命题正确但执行不到位 → 坚持，换执行                        │
│  □ 有边界冲突但有解决方案 → 坚持，找方案                          │
│  □ 团队信心动摇但方向正确 → 坚持，给时间                          │
│                                                                     │
│  【什么时候放弃？】                                                │
│  □ 关系命题本身错误 → 放弃，重新定义                              │
│  □ 边界不可调和 → 放弃，尊重边界                                  │
│  □ 合作方不ready → 放弃，等待时机                                  │
│                                                                     │
│  【艾维的判断标准】[B]                                             │
│  "坚持不是因为我们已经投入，而是因为方向正确"                        │
│  "放弃不是因为难，而是因为关系命题错了"                            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.2 案例：坚持 vs 放弃时刻

```
【坚持案例】Apple iPhone 开发 [B]
- 初期：团队对多点触控不确定
- 艾维坚持：关系命题（人与内容的无障碍关系）正确
- 结果：等待技术成熟，最终成功

【放弃案例】某个被取消的Apple项目 [B]
- 艾维的判断：关系命题无法在现有技术下正确实现
- 决定：放弃，等待更好的时机
- 教训：不是所有想法都能现在实现

【时机教训】
→ 好的想法 + 错误的时机 = 失败
→ 一般的想法 + 正确的时机 = 成功
→ 好的想法 + 正确的时机 + 坚持执行 = 卓越
```

## 4. 环境因素评估

```
萃取/协作前必须评估：

1. 专家/合作方状态
   ├── 时间充裕度
   ├── 表达意愿
   ├── 信任程度
   └── 情绪状态

2. 组织环境
   ├── 知识分享文化
   ├── 萃取激励机制
   ├── 保密敏感性
   └── 萃取优先级

3. 领域特征
   ├── 知识可编码程度
   ├── 时效性要求
   ├── 风险敏感度
   └── 案例丰富度

4. 市场时机
   ├── 行业成熟度
   ├── 竞争格局
   ├── 技术可实现性
   └── 用户接受度
```

## 5. 五维自检清单

```
在每次萃取/协作前，检查五维是否到位：

【核心理念】
□ 明确萃取/协作的使命
□ 理解隐性知识的价值
□ 坚持长期主义

【规则体系】
□ SECI模型指导萃取方法
□ 萃取对象评估完成
□ 边界清单确认

【工具基建】
□ CPSO格式准备
□ 访谈模板准备
□ 验证工具准备

【执行技巧】
□ 五步萃取法执行
□ 追问到判断逻辑
□ 收集反例和边界

【环境时机】
□ 时机合适
□ 专家/合作方状态良好
□ 环境支持萃取/协作

【十年测试】
□ 知识/设计十年后仍有效吗？
□ 知识/设计十年后能理解吗？
□ 知识/设计十年后能更新吗？
```
