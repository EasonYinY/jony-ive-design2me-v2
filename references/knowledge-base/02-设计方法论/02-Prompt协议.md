> **版本**：1.0
> **日期**：2026-06-22

## 0. 协议概述

### 0.1 为什么需要Prompt协议

```
┌─────────────────────────────────────────────────────────────────────┐
│                    方法论 → AI生成的桥梁                             │
├─────────────────────────────────────────────────────────────────────┤
```

## 1. 第一层：关系定义层

### 1.1 关系命题模板

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
核心模板
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

This is not a [default category];
it is a relationship between [person],
[action], [environment], and [trust].

这不是什么品类；
这是什么人通过什么行为、在什么环境下，建立什么信任的关系。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
模板要素详解
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【person - 用户】
- 谁在使用这个产品？
- 用户的背景、需求、期望？
- 用户与产品建立什么关系？

【action - 行为】
- 用户需要做什么？
- 什么行为是高频的？
- 什么行为需要身体确认？
- 用精确动词描述：extinguish vs switch off

【environment - 环境】
- 产品在什么环境下使用？
- 环境有什么约束？
- 环境需要什么特性？

【trust - 信任】
- 产品要建立什么信任？
- 长期陪伴？可靠？诚实？
- 信任的证据是什么？

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
关系命题示例

产品：便携灯具
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
❌ 错误关系命题：
"这是一个现代风格的台灯"

✓ 正确关系命题：
"This is not a desk lamp;
it is a relationship between a person
and light, carrying, extinguishing,
and maritime environment for trust."

"这不是一个台灯；
这是人与光、携带、熄灭、海洋环境的关系，
建立的是可靠陪伴的信任。"

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

产品：智能手表
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
❌ 错误关系命题：
"这是一个智能手表"

✓ 正确关系命题：
"This is not a computer on your wrist;
it is a relationship between a person

for confident presence."

"这不是手腕上的电脑；
这是人与身体、时间、健康、身份确认的关系，
建立的是自信存在的信任。"

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

产品：无线耳机
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
❌ 错误关系命题：
"这是一个无线耳机"

✓ 正确关系命题：
"This is not a pair of earbuds;
it is a relationship between a person
and music, wireless ritual, and disappearing hardware
for uninterrupted listening."

"这不是一副耳机；
这是人与音乐、无线仪式、消失型硬件的关系，
建立的是不被打断的聆听信任。"
```

### 1.3 行为动词表

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
为什么行为动词比名词重要
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

❌ 名词思维：
"设计一个灯" → 想到台灯造型 → 形态受限

✓ 动词思维：
"设计一个让用户extinguish、carry、hang的产品"
→ extinction、carrying、hanging → 形态从行为推导

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
艾维核心行为动词表
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

┌────────────────────────────────────────────────────────────────────┐
│                    行为动词对 · 中文对照                             │
├────────────────────────────────────────────────────────────────────┤
│                                                                    │
│  extinguish  vs  switch off                                      │
│  熄灭（蜡烛/灯）      vs  关灯（电子设备）                        │
│  → 仪式感更强，暗示需要旋钮/动作                                  │
│                                                                    │
│  carry       vs  hold                                            │
│  携带/背负           vs  握住                                    │
│  → 需要悬挂结构，不是简单手柄                                      │
│                                                                    │
│  hang        vs  place                                           │
│  悬挂于某处          vs  放在桌上                                 │
│  → 需要悬挂点，不是平面底座                                        │
│                                                                    │
│  tune        vs  adjust                                          │
│  调校/精调           vs  调节                                    │
│  → 暗示精确性，需要档位反馈                                       │
│                                                                    │
│  unlock      vs  open                                            │
│  解锁开启           vs  开门                                       │
│  → 暗示安全确认，需要触感反馈                                      │
│                                                                    │
│  seal        vs  close                                           │
│  封缄/密封           vs  关闭                                    │
│  → 暗示密封性，需要确认机制                                        │
│                                                                    │
│  listen      vs  play                                            │
│  聆听仪式           vs  播放音乐                                  │
│  → 暗示沉浸感，需要仪式化设计                                      │
│                                                                    │
│  drive       vs  operate                                          │
│  驾驭仪式           vs  操作机器                                  │
│  → 暗示人机关系，需要身体确认                                      │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
行为动词选择流程
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Step 1: 列出产品涉及的所有行为
├── 用户会用这个产品做什么？
├── 什么动作是高频的？
└── 什么动作需要精确控制？

Step 2: 将动词升级为精确动词
├── "关灯" → "extinguish"
├── "拿东西" → "carry" 或 "grasp"
└── "听音乐" → "listen"

Step 3: 基于精确动词推导形态
├── extinguish → 需要熄灭动作确认（旋钮？开关？）
├── carry → 需要携带结构（织带？手柄？）
└── listen → 需要聆听仪式（角度？姿态？）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
行为动词速查表
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

| 场景 | 通用词 | 精确词 | 形态暗示 |
|------|--------|--------|----------|
| 照明 | 开关灯 | extinguish | 仪式感熄灭动作 |
| 携带 | 拿东西 | carry | 悬挂/背负结构 |
| 悬挂 | 放上去 | hang | 悬挂点设计 |
| 调节 | 调一下 | tune | 精确档位反馈 |
| 开启 | 打开 | unlock | 安全确认触感 |
| 关闭 | 关上 | seal | 密封确认 |
| 聆听 | 听音乐 | listen | 仪式化姿态 |
| 驾驶 | 开车 | drive | 人机身体确认 |
| 充电 | 充电 | refuel | 能量补充仪式 |
| 校准 | 校准 | calibrate | 精确调整动作 |
```

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
材料职责的核心原则
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

❌ 错误写法：
"使用高级金属材质"
"采用奢华皮革包装"
"水晶般透明质感"

✓ 正确写法：
"[材料] for [功能职责]"
"[材料] for [情绪品质]"
"[材料] for [时间老化]"

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
材料职责三问
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Q1: 功能职责
这个材料在这个产品中做什么物理/化学/技术功能？

例：precision-ground glass for light diffusion
→ 玻璃承担光扩散的物理功能

例：brushed stainless steel for salt corrosion resistance
→ 不锈钢承担防盐腐蚀的化学功能

Q2: 情绪品质
这个材料传达什么情绪/感受？

例：brushed metal for trust and precision
→ 拉丝金属传达信任和精确感

例：warm wood for domestic comfort
→ 温润木材传达家居舒适感

Q3: 时间老化
这个材料十年后会变成什么样？

例：brushed metal for aging gracefully
→ 拉丝金属划痕会成为好看的包浆

例：natural leather for developing patina
→ 真皮会形成有故事的老化痕迹

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

```
艾维说："手是最诚实的感官"

❌ 视觉可以被欺骗
✓ 手会告诉你真相

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

├── 高频行为需要触觉确认
├── 用户需要"知道"操作生效

├── 声音（sound）→ 咔嗒声

├── 手指如何找到触觉点？
├── 形状是否提供方向引导？
└── 是否需要盲操作能力？

Q4: 高频还是低频？
├── 高频（每天多次）→ 需要快速确认
├── 低频（偶尔）→ 需要精确确认
└── 这决定了反馈形式的选择
```

## 4. 第四层：验证标准层

### 4.1 关系验证

```
在输出设计后，必须验证：

□ 能用一句话说清楚这是什么关系吗？
"这不是一个[默认品类]，这是[用户]与[行为]、[环境]、[信任]的关系。"

□ 形态是否服务于关系？
├── 形态从关系推导出来了吗？
├── 形态能证明关系存在吗？
└── 形态会强化关系吗？

□ 关系命题中的每个要素是否都有对应的设计决策？

如果说不清楚 → 重写关系命题
如果形态不服务关系 → 重新推导形态
```

### 4.2 职责验证

```
每个材料必须能回答：

Q1: 这个材料在这里做什么功能？
Q2: 这个材料传达什么情绪？
Q3: 这个材料十年后会变成什么样？

每个触觉节点必须能回答：

Q1: 这个触觉确认什么行为？
Q2: 反馈形式是什么？
Q3: 位置设计合理吗？
Q4: 是高频还是低频？

□ 产品有3个以上触觉确认节点吗？
如果少于3个 → 增加触觉节点
```

### 4.3 边界验证

```
问：这个产品十年后还能被维护吗？

□ 可维修吗？□ 可清洁吗？
□ 能老化得好看吗？□ 可更换吗？

如果不能 → 重新考虑可维护性

□ 知道什么在"让位"吗？
□ 引用了行为/秩序，而非外形？
□ 有 not nostalgic replica 约束？
```
