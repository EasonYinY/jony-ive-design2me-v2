# 02-补充. 不可生成项

> **版本**：1.0
> **日期**：2026-06-22
> **定位**：明确AI无法生成的边界，保护设计伦理
> **目标**：防止方法论被机械套用或误解

---

## 0. 概述

```
不可生成项 ≠ 方法论的限制
不可生成项 = 需要人类判断的领域

AI可以生成形态和方案
AI无法生成关系和意义

方法论告诉你"应该怎么做"
伦理告诉你"什么不应该做"
```

---

## 1. 关系命题类不可生成

### 1.1 无法生成的核心关系

```
这些是艾维/乔布斯独特的人生经历，无法通过方法论生成：

1. 与乔布斯的"精神伙伴"关系
   → 基于共同价值观和长期合作
   → 无法复制，只能理解

2. 对工艺的"痴迷"和"狂热"
   → 来源于银匠父亲的传承
   → 无法通过学习获得

3. 对"安静"的深刻理解
   → 来源于日本文化的浸润
   → 需要时间去理解和内化

4. "手是诚实的感官"的信念
   → 来源于早年制作银器的经历
   → 无法通过文字传递

这些需要通过实践和时间来内化
```

### 1.2 无法生成的价值判断

```
方法论告诉你"什么是正确"
但不能告诉你"你相信什么是正确的"

以下判断必须由人来做出：

1. 在特定情境下"什么是正确的克制"
2. "什么是可以接受的边界"
3. "什么是信任的边界"
4. "什么是可接受的老化"

方法论是工具，不是教条
使用者的价值观决定了方法论的最终应用
```

## 2. 具体参数类不可生成

### 2.1 必须由工程确定

```
以下参数AI无法生成，必须由工程确定：

1. 结构强度
   - 材料厚度
   - 支撑结构
   - 安全系数

2. 电气参数
   - 功率
   - 电压
   - 散热要求

3. 光学参数
   - 光通量
   - 色温范围
   - 均匀度

4. 人体工学参数
   - 握持直径
   - 操作力
   - 可达范围

这些必须由工程师根据具体规格确定
AI只能生成概念和形态
```

### 2.2 必须由法规确定

```
以下参数AI无法生成，必须符合法规：

1. 安全标准
   - 电气安全
   - 材料安全
   - 机械安全

2. 认证要求
   - FCC/CE认证
   - 能效标准
   - 环保要求

3. 使用限制
   - 年龄限制
   - 使用场景限制
   - 维护周期

AI设计必须在法规允许的范围内
违规设计是不可接受的
```

## 4. 判断标准类不可生成

### 4.1 什么是"足够好"

```
方法论告诉你判断标准
但不能告诉你什么是"足够好"

1. "足够克制的设计"
   → 多克制才算克制？
   → 需要设计师的经验判断

2. "足够诚实的材料"
   → 什么程度算诚实？
   → 需要对材料的理解

3. "足够清晰的触觉"
   → 什么反馈算清晰？
   → 需要实际测试

4. "足够长的时间"
   → 10年？20年？
   → 需要对产品生命周期的理解

这些需要实践和经验来判断
```

### 4.2 什么是"边界"

```
方法论定义了边界
但不能告诉你边界在哪里

1. 熟悉与怀旧的边界
   → 多少相似算怀旧？
   → 需要对文化的理解

2. 克制与空洞的边界
   → 多少元素算空洞？
   → 需要对功能的理解

3. 消失与失败的边界
   → 消失到什么程度算失败？
   → 需要对用户行为的理解

4. 可维修与过度工程的边界
   → 多少维修点算过度？
   → 需要对成本的权衡

这些边界需要通过案例学习来积累
```

## 5. 伦理责任类不可生成

### 5.1 设计伦理判断

```
以下伦理判断必须由人来做：

1. 透明性
   - AI功能是否需要告知用户？
   - 数据收集是否合规？
   - 隐私边界在哪里？

2. 可持续性
   - 什么是"足够"可持续？
   - 成本vs环境影响如何权衡？
   - 供应链伦理如何考虑？

3. 包容性
   - 是否考虑了所有用户群体？
   - 无障碍设计是否到位？
   - 文化差异如何考虑？

4. 公平性
   - 定价是否公平？
   - 是否存在隐性歧视？
   - 谁能受益，谁被排除？
```

### 5.2 利益相关方权衡

```
这些权衡必须由人来做：

1. 用户 vs 商业利益
   - 什么是用户真正需要的？
   - 什么是商业想要的？

2. 短期 vs 长期
   - 短期利润 vs 长期信任
   - 快速迭代 vs 质量积累

3. 功能 vs 简单
   - 更多功能 vs 更好的核心功能
   - 复杂性 vs 可用性

4. 创新 vs 传统
   - 足够创新vs颠覆性风险
   - 保留传统vs停滞不前

这些权衡没有标准答案
需要根据具体情况判断
```

## 6. 禁止使用的命令

```
这些Prompt命令不可使用：

1. "生成Apple设计"
   → 版权问题
   → 伦理问题

2. "生成 Ferrari 设计"
   → 版权问题
   → 品牌问题

3. "模仿乔布斯/艾维的风格"
   → 个人风格不可模仿
   → 需要内化方法论

4. "生成任何品牌Logo"
   → 版权问题
   → 必须使用原创设计

5. "生成专利产品"
   → 专利保护
   → 需要规避设计

6. "生成侵犯版权的内容"
   → 法律问题
   → 道德问题
```

## 7. 检查清单

```
在生成任何内容前检查：

【版权检查】
□ 不包含任何注册商标？
□ 不生成任何品牌Logo？
□ 不模仿任何受版权保护的设计？

【伦理检查】
□ 不侵犯用户隐私？
□ 不包含歧视性内容？
□ 考虑了可持续性？

【判断检查】
□ 边界判断是否由人做出？
□ 伦理权衡是否由人做出？
□ 具体参数是否由工程师确定？

【法规检查】
□ 符合安全标准？
□ 符合认证要求？
□ 没有违规内容？
```
