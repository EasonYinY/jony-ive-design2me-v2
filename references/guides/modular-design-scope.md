# 模块化设计适用场景限定

> **版本**: 1.0
> **日期**: 2026-06-21
> **触发**: R1评审发现双胶囊模块化设计评分仅62/90，模块化增加复杂度但实用性不足

---

## 核心原则

**模块化不是万能药**。模块化设计有其明确的适用场景和不适用场景，错误使用模块化会导致：
1. 复杂度增加（多部件+连接机构）
2. 可靠性降低（连接点故障）
3. 成本上升（多模具+装配）
4. 识别性分散（非统一整体）

## 模块化适用场景

| 场景 | 示例 | 模块化价值 | 评分参考 |
|------|------|-----------|---------|
| **多单元组合需求明确** | 耳机（左右耳）、电池组 | 用户需要灵活组合 | 80+/90 |
| **功能扩展需求** | 相机镜头、工具头 | 用户需要更换功能模块 | 75+/90 |
| **多用户共享** | 积木、拼图 | 多人协作创造 | 70+/90 |
| **生命周期差异** | 手机壳、表带 | 部分频繁更换 | 75+/90 |

## 模块化不适用场景

| 场景 | 示例 | 模块化问题 | 评分参考 |
|------|------|-----------|---------|
| **单功能产品** | 灯具、充电器、音箱 | 无组合需求，增加复杂度 | 60-65/90 |
| **高可靠性要求** | 户外装备、医疗设备 | 连接点故障风险 | 55-65/90 |
| **极简美学要求** | 艾维级设计 | 连接缝破坏整体感 | 60-70/90 |
| **成本敏感** | 大众消费品 | 多模具+装配成本 | 55-65/90 |

## 实证数据

### Round 1 户外露营灯

| 方向 | 模块化程度 | 评分 | 问题 |
|------|-----------|------|------|
| D1 | 无（单体） | 65/90 | 形态俗套 |
| D2 | 无（单体） | **83/90** | 无模块化问题 |
| D3 | 高（双胶囊+连接桥） | 62/90 | 连接桥破坏整体感 |

**关键发现**: 户外灯作为单功能产品，模块化无价值，反而降低评分 **-21分**

## 决策树

```
是否需要模块化？
├── 用户是否需要组合/扩展/更换？
│   ├── 是 → 继续评估
│   └── 否 → 不使用模块化
├── 可靠性要求是否极高？
│   ├── 是 → 不使用模块化
│   └── 否 → 继续评估
├── 美学要求是否极简/整体？
│   ├── 是 → 不使用模块化
│   └── 否 → 可使用模块化
└── 成本是否敏感？
    ├── 是 → 不使用模块化
    └── 否 → 可使用模块化
```

## 检查清单

在STEP-06生成候选时，增加以下检查：

```
[ ] 该产品是否需要多单元组合？
[ ] 该产品是否需要功能扩展？
[ ] 该产品的可靠性要求是否极高？
[ ] 该产品的美学要求是否极简？
[ ] 该产品的成本是否敏感？
[ ] 若使用模块化，连接机构是否可靠？
[ ] 若使用模块化，连接缝是否破坏整体感？
```

## 替代方案

当模块化不适用时，考虑以下替代方案：

| 替代方案 | 适用场景 | 示例 |
|---------|---------|------|
| **一体化设计** | 单功能产品 | 鹅卵石形态灯具 |
| **隐藏式功能** | 多功能但需简洁 | 滑动调光、触摸开关 |
| **配件化** | 偶尔需要扩展 | 可拆卸挂钩、底座 |
| **软件模块化** | 功能差异 | APP调节、固件升级 |
