# 形态比例与部件层级

> **版本**: 1.1
> **日期**: 2026-06-21
> **触发**: 用户反馈 mouse nextgen "有机造型无规则，无基础形态，像学生作品"

---

## 核心原则

形态必须由**基础体块**出发，通过**理性变形规则**导出。任何直接从"未来感""有机""流线型"开始的形态推导均视为失败。

## 基础形态锁定法

### 原理

优秀提示词不是从"有机曲线"开始，而是先定义**基础体块**，再写**变形规则**。基础体块是形态的"骨架"，变形规则是"肌肉"，没有骨架的肌肉就是"乱七八糟的有机"。

### 基础形态词库

| 基础形态 | 英文 | 适用产品 | 变形方向 | 姿态词 |
|----------|------|----------|----------|--------|
| **板状** | slab | 鼠标、咖啡机、灯具 | 弯曲、倾斜、挖空 | Floats |
| **块状** | block | 音箱、充电器、工具 | 倒角、分割、层叠 | rests on / Floats |
| **圆柱** | cylinder | 灯具、容器、笔 | 锥化、扭曲、挖空 | suspended / Floats |
| **圆顶** | dome | 鼠标、控制器、盖子 | 压扁、偏移、切割 | nestles / Floats |
| **楔形** | wedge | 支架、底座、工具 | 倾斜、弯曲、分割 | rests on |
| **圆盘** | disc | 旋钮、底座、托盘 | 凹陷、边缘处理 | Floats |

### 基础形态锁定步骤

```
Step 1: 选择基础形态
  → 从词库中选择1个最符合产品功能的基础体块
  → 例：鼠标 → slab（手掌平放）

Step 2: 定义精确尺寸
  → 长×宽×高，精确到小数点后1位
  → 例：120mm × 65mm × 35mm

Step 3: 定义变形规则
  → 从人体工学/功能/美学出发，定义如何变形
  → 例：前宽后窄（手掌支撑），6°前倾（手腕自然角度）
  → 变形必须有数学/物理依据，不能随意

Step 4: 定义几何术语
  → 使用G2/G1连续、fillet、chamfer、tangent、step等术语
  → 例：G2 fillet edges, sweeping tangent base

Step 5: 验证
  → 是否从基础体块出发？
  → 变形是否有理性依据？
  → 是否包含至少2个精确数字？
```

### 失败检测

- ❌ 直接从"有机曲线""流线型""未来感"开始
- ❌ 无基础体块定义
- ❌ 变形无数学/物理依据
- ❌ 缺少精确数字
- ❌ 缺少几何术语

## 理性-有机转换规则

### 原理

有机曲线不是随意绘制，而是从**理性基础**通过**数学变换**生成。每个曲面必须有明确的来源和依据。

### 转换公式

```
有机形态 = 基础体块 + 变形规则 + 人体工学偏移 + 材料约束

例：
鼠标掌托 = slab（基础）
         + 前宽后窄（变形规则：手掌支撑）
         + 6°前倾（人体工学：手腕自然角度）
         + G2 fillet（材料约束：碳纤维可成型曲率）
```

### 人体工学数据锚点（鼠标）

| 参数 | 数值 | 来源 | 形态影响 |
|------|------|------|----------|
| 手掌宽度 | 75-95mm | 人体测量 | 鼠标宽度 65-75mm |
| 握持直径 | 45-55mm | 人体测量 | 掌托高度 35-45mm |
| 掌托 | 70mm | 人体工学 | 后缘支撑位置 |
| 前倾角度 | 6-12° | 人体工学 | 手腕自然角度 |
| 滚轮位置 | 居中偏后5mm | 操作习惯 | 食指自然落点 |
| 左右按键间隙 | 0.1mm | 制造精度 | 触感反馈 |
| 掌托凹陷 | 0.3mm | 人体工学 | 掌心贴合 |

### 曲面来源标注规则

每个曲面必须标注来源：

```
曲面名称 + 来源类型 + 具体依据

例：
- "掌托曲面" + "人体工学偏移" + "从手掌握持曲线偏移3mm"
- "侧握曲面" + "功能约束" + "拇指凹槽深度2mm，防滑"
- "顶面曲面" + "材料约束" + "碳纤维G2连续可成型曲率"
```

## 原内容（保留）

形态必须由身体、受力、构件布置和操作顺序导出。`form_proportion` 记录主次体量、轮廓连续性、转折位置、接缝秩序和关键构件尺度，并说明每项如何对应结构或动作；任何仅为制造“未来感”的附加曲面、透明件或灯带均不得进入结论。

`resting_state` 单独记录停止使用或断电后的构件位置、重心稳定、状态可读性、收纳与防误触关系。工作状态成立但静息状态依赖悬浮、隐藏支撑或持续通电时不得通过。

评审时先检查产品是否可识别，再检查比例是否表达构件职责；摄影不能掩盖比例冲突。
