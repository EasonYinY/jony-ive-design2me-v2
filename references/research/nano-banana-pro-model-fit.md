## 第一节：模型身份与架构

### 1.1 模型定位

**Nano Banana Pro** 是 Lovart 平台部署的**写实级扩散模型（Diffusion Model）**，基于 Google Gemini 3 Pro Image 架构，带 Think Mode 推理能力。

| 属性 | 说明 |
|------|------|
| **底层架构** | Diffusion Model + Gemini 3 Pro 推理引擎 |
| **训练集** | 工业设计渲染图、产品摄影、建筑可视化、汽车设计草图 |
| **优势领域** | 材质表面质感、光影物理正确性、空间方位关系 |
| **劣势领域** | 时间跨度理解、功能职责解析、内部制造工艺、动态叙事 |
| **典型输出** | 1024×1024 / 2048×2048 产品渲染图 |

### 1.2 与消费级模型的差异

| 维度 | Nano Banana Pro | Midjourney v6 | DALL-E 3 | Flux 1.0 Pro |
|------|-----------------|---------------|----------|--------------|
| **材质质感** | ⭐⭐⭐⭐⭐ 极高 | ⭐⭐⭐⭐ 高 | ⭐⭐⭐ 中 | ⭐⭐⭐⭐ 高 |
| **几何精确性** | ⭐⭐⭐⭐ 高 | ⭐⭐⭐ 中 | ⭐⭐ 低 | ⭐⭐⭐⭐ 高 |
| **空间关系** | ⭐⭐⭐⭐⭐ 极高 | ⭐⭐⭐⭐ 高 | ⭐⭐⭐ 中 | ⭐⭐⭐⭐ 高 |
| **语义遵循度** | ⭐⭐⭐⭐ 高（带 Think Mode） | ⭐⭐⭐ 中 | ⭐⭐⭐⭐ 高 | ⭐⭐⭐⭐ 高 |
| **微观尺寸** | ⭐⭐ 低（<1mm 丢失） | ⭐⭐⭐ 中 | ⭐⭐ 低 | ⭐⭐⭐ 中 |
| **工程词汇** | ⭐ 极低（噪音化） | ⭐⭐ 低 | ⭐⭐ 低 | ⭐⭐ 低 |
| **训练集先验** | 强（金属默认 brushed） | 强（科幻默认 glowing） | 中 | 中 |

## 第二节：官方文档与明示能力边界

### 2.1 Google 官方明示（Gemini 3 Pro Image 基础）

**来源**: Google Cloud 官方文档 + Google AI 博客 + Gemini 3 Pro 技术报告

| 能力 | 官方说明 | 对提示词工程的影响 |
|------|----------|-------------------|
| **推理模式（Think Mode）** | "优先理解意图而非字面参数" | 给"意图"比给"精确数字"更有效 |
| **图像生成上限** | 2048×2048，支持多纵横比 | 产品渲染足够，建筑/场景可能不足 |
| **文本渲染能力** | "不支持精确文本渲染" | 禁止提示词中包含文字/logo/标识 |
| **数字遵循度** | "不保证精确数字执行" | 数字仅作为意图暗示，非硬约束 |
| **微观细节** | "<1mm 级细节不可控" | 宏观产品必须泛化微观尺寸 |
| **材质库** | 训练集包含大量工业设计摄影 | 对"brushed titanium""smoked glass"有稳定响应 |

### 2.2 Lovart 平台特定约束

**来源**: Lovart 官方 API 文档 + 开发者论坛

| 约束 | 说明 | 应对策略 |
|------|------|----------|
| **超时机制** | 180s 生成超时，静默失败 | 后台运行 + 轮询检查 |
| **成功检测** | 不返回明确 success 状态 | 必须图片下载验证 |
| **并发限制** | 单账户 2-3 并发 | 串行生成或限制并发 |
| **输出格式** | PNG only，无 EXIF | 外部记录生成参数 |
| **风格漂移** | 10-20% 概率完全忽略 prompt | 多轮生成取最优 |

### 3.1 ppl.studio 1000+ 张实测（2026-06-21~23）

**来源**: 本 skill 历史运行记录 + 用户评审反馈

| 测试项 | 样本量 | 结果 | 结论 |
|--------|--------|------|------|
| 词数 sweet spot | 200+ 张 | 40-80 words 最优 | <40 under-specifying，>100 互相矛盾 |

| 摄影参数位置 | 100+ 张 | 第 5 段优于第 1 段（Google 官方公式对齐） | 摄影锚放 Style 段 |

## 第五节：模型适配策略矩阵

### 5.1 按产品尺度的策略

| 产品尺度 | 数字精度 | 微观处理 | 姿态词 | 典型示例 |
|---------|---------|---------|--------|----------|
| **微观**（<20cm） | 0.1mm | 保留 | Floats / levitates | 鼠标、耳机、手表 |
| **中观**（20cm-1m） | 1mm | 保留 ≤2 个 | Floats / rests on | 咖啡机、灯具、音箱 |
| **宏观**（>1m） | 1cm | 泛化为视觉信号 | Floats / suspended | 飞行器、家具、载具 |

### 5.2 按模型特性的策略

| 模型特性 | 应对策略 | 对应方法/协议 |
|---------|---------|-------------|
| **优先理解意图** | 给感官描述而非工程参数 | 方法 17 视觉等效洗涤 |
| **<1mm 丢失** | 宏观产品泛化微观尺寸 | 方法 18 尺度缩放滤网 |
| **工程词汇噪音** | 过滤 lifespan/manufacturing 等 | compile_prompt.py 噪音词过滤 |
| **训练集先验强** | 强制使用词库绕过默认 | 方法 16 决策树 + 词库 §3/§4/§5 |
| **10-20% 忽略 prompt** | 多轮生成取最优 | 终结交付步 3 张图 |
| **水面漂浮幻觉** | 用 Light interaction 替代 Floats | 五段式 Block 4 升级 |
