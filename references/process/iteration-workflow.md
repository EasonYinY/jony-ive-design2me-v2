# 迭代工作流（多轮升级模式）

> **版本**: 1.0
> **日期**: 2026-06-21
> **触发**: 用户要求执行多轮迭代（如"执行5轮""迭代升级"）

---

## 什么是迭代模式

迭代模式是在标准十五步流程基础上，增加图片生成、评审、升级、Handoff 的完整闭环。

**与单次模式的区别**:

| 模式 | 终点 | 图片生成 | 评审 | 升级 | Handoff |
|------|------|---------|------|------|---------|
| 单次模式 | STEP-15 提示词 | 需用户明确指令 | 无 | 无 | 无 |
| 迭代模式 | 技能升级+Handoff | 自动执行 | 8维度+艾维评估 | 自动升级 | 自动生成 |

---

## 迭代流程

### Phase 0: 准备阶段

1. 制定总迭代计划（如"5轮迭代计划"）
2. 确定每轮产品（随机或指定）
3. 创建 `references/iterations/5-round-iteration-plan.md`

### Phase 1: 单轮执行（每轮独立）

每轮包含以下步骤：

```
STEP 1: 创建迭代计划
  → 写入 references/iterations/round-N/iteration-plan.md

STEP 2: 执行15步推理
  → 写入 references/iterations/round-N/design-output.md

STEP 3: Lovart出图（3方向并行）
  → 输出至 ~/lovart-out/rN-d{1-3}/
  → 使用 vertex/anon-bob (Nano Banana Pro) 模型
  → 1K分辨率，纯白背景

STEP 4: Vision分析+艾维评估
  → 写入 references/iterations/round-N/image-analysis.md
  → 8维度评审 + 艾维水平评分
  → 参考知识库：设计师-知识蒸馏/乔纳森·艾维-知识蒸馏

STEP 5: 生成升级计划
  → 写入 references/iterations/round-N/upgrade-plan.md
  → 确定需要升级的技能文件

STEP 6: 执行升级
  → 新增/修改 references/ 下的文件
  → 更新版本记录 references/changelogs/vX.Y.Z-changelog.md

STEP 7: 生成Handoff口令
  → 写入 references/iterations/handoff-rN-rN+1.md
  → 包含上轮关键发现、技能版本、下轮启动指令
```

### Phase 2: 轮次切换

1. **独立性原则**: 每轮完全独立，不共享上下文
2. **新窗口/代理**: 通过 Handoff 口令启动下一轮
3. **技能版本**: 每轮使用当时最新技能版本

### 复杂产品分两次生成策略（当前版本+）

对于高复杂度产品（咖啡机、相机、复杂工具），采用分两次生成策略：

**第一次**: 主体形态 + 核心材料（300-350词）
**第二次**: 细节补充 + 交互细节（同一 thread-id，150-200词）

**适用条件**:
- 产品包含3个以上独立部件
- 需要描述机械结构（折叠、旋转、伸缩）
- 预期评分低于8.0时

**验证方法**:
```
[ ] 产品复杂度已评估（部件数≥3？）
[ ] 材料选择偏向自然/稳定材料（陶瓷、玻璃、石材）
[ ] 机械结构描述已精简（用"状态指示器"替代详细机构）
[ ] 预期评分已调整（复杂产品7.0-8.0）
[ ] 已准备分两次生成（第一次300-350词，第二次150-200词）
```

## 艾维水平评估标准

### 评分维度（8维度）

| 维度 | 权重 | 说明 |
|------|------|------|
| 形态比例与体量感 | 15% | 比例关系、体量感、视觉张力 |
| 材料质感与CMF | 15% | 材料诚实、质感表达 |
| 接缝秩序与制造精度 | 15% | 接缝精密、制造感 |
| 人体工学合理性 | 15% | 尺寸、角度、力度 |
| 识别性与品牌特征 | 15% | 独特性、记忆点 |
| 静息状态表达 | 15% | 静息形态、收纳友好 |
| 背景纯净度 | 5% | 纯白背景、无干扰 |
| 与艾维设计对比 | 5% | 艾维核心16法对照 |

### 评分等级

| 等级 | 分数 | 描述 |
|------|------|------|
| 艾维级 | 80-90/90 | LoveFrom水平，可量产 |
| 准艾维级 | 65-79/90 | 接近，需优化 |
| 普通级 | 50-64/90 | 工业设计水平 |
| 未达标 | <50/90 | 未达标准 |

### 产品复杂度与预期评分

基于5轮迭代实证，不同复杂度产品的预期评分：

| 产品复杂度 | 典型产品 | 预期评分 | 艾维级可能性 | 关键策略 |
|-----------|---------|---------|-------------|---------|
| 低 | 无线充电器、桌面灯具 | 8.5-9.0 | **高** | 优先自然形态+材料对比 |
| 中 | 智能手表、便携音箱 | 7.5-8.5 | 中 | 控制机械结构描述，强化材料质感 |
| 高 | 咖啡机、相机、复杂工具 | 7.0-8.0 | 低 | 分两次生成，优先陶瓷/玻璃等稳定材料 |

**关键发现**：自然形态（鹅卵石、水滴）比机械形态更易达到艾维级，因为AI对有机形态的理解更稳定。

### 5轮迭代评分演进（旅行户外灯实证）

基于2026-06-22执行的5轮25代理75方案迭代：

| 轮次 | 技能版本 | 关键改进 | 最低分 | 最高分 | 平均分 |
|------|----------|----------|--------|--------|--------|
| Round 1 | 当前版本 → 当前版本 | 基础框架 + 4材料对比 + 隐性文化杂交 | 64/90 | 82/90 | 75.4 |
| Round 2 | 当前版本 → 当前版本 | 方向同质化修复 + 操作/材料多样化 | 78/90 | 83/90 | 80.2 |
| Round 3 | 当前版本 → 当前版本 | 工艺多样化 + 验证差异化 | 80/90 | 84/90 | 82.0 |
| Round 4 | 当前版本 → 当前版本 | 情感连接 + 突破85分 | 81/90 | 85/90 | 83.2 |
| Round 5 | 当前版本 → 当前版本 | 故事化设计 + 达到87分 | 83/90 | 87/90 | 84.6 |

**关键发现**：
1. 4材料对比是基线（所有高分方案≥80/90均使用）
2. 隐性文化杂交优于显性（规避显性文化符号，评分更高）
3. 单一体块优先（简化结构，提升制造精度）
4. 三维差异化是突破80分关键（操作/材料/工艺）
5. 情感连接是突破85分关键
6. 故事化设计是达到87分的关键

## 升级策略

### 升级类型优先级

| 优先级 | 类型 | 说明 |
|--------|------|------|
| 🔴 高 | 流程缺陷 | 影响设计质量的方法论缺失 |
| 🟡 中 | 方法论缺失 | 特定品类/场景的设计方法 |
| 🟢 低 | 表达优化 | 提示词、评审维度优化 |
| 🔵 低 | 评审增强 | 图片评审维度扩展 |

### 升级文件类型

| 类型 | 存放位置 | 命名规范 |
|------|---------|---------|
| 指南 | `references/guides/` | `[topic].md` |
| Pitfall | `references/cases/` | `pitfall-NNN-[name].md` |
| 流程补丁 | `references/process/` | 修改现有文件 |
| 质量门补丁 | `references/quality/` | 修改现有文件 |
| 版本记录 | `references/changelogs/` | `vX.Y.Z-changelog.md` |

## 注意事项

1. **独立性**: 每轮迭代完全独立，不共享上下文
2. **技能版本**: 每轮使用当时最新技能版本
3. **文件卫生**: 所有升级内容写入技能目录，不污染主文件
4. **版本记录**: 每轮升级必须记录 changelog
5. **Handoff**: 每轮结束必须生成 Handoff 口令
6. **Subagent 限流**: 当并行启动5个 `delegate_task` 时，频繁触发 HTTP 429 限流。推荐主会话直接执行，或最多2个并行
