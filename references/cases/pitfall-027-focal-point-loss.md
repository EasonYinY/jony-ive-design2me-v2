# Pitfall 027: 焦点特征丢失

> **版本**: 1.0
> **日期**: 2026-06-21
> **触发**: Round 3 D2涟漪波纹中心凹陷在图片中不够明显，导致识别性扣分

---

## 现象

提示词中明确描述的"焦点特征"（如中心凹陷、关键纹理、标志性细节）在生成的图片中被弱化、平滑化或完全丢失。

## 根因

1. **模型注意力分散**: 提示词包含多个特征时，模型优先渲染整体印象
2. **细节尺度问题**: 小尺度细节（如0.3mm凹陷）在图片中难以表达
3. **语义平滑**: 模型倾向于"美化"表面，平滑化纹理和凹陷
4. **提示词权重不足**: 焦点特征在提示词中只出现一次，权重不足

## 影响

- 识别性评分降低（D2从预期的9/10降至实际的8/10）
- 设计意图无法完整传达
- 艾维总分距目标差1-2分

## 解决

参考 `references/guides/focal-point-emphasis.md`：

1. **重复强调法**: 焦点特征在提示词中出现≥2次
2. **对比强调法**: 通过明暗/材质对比突出焦点
3. **尺度强调法**: 明确焦点的相对尺度
4. **位置强调法**: 明确焦点的精确位置

## 检查清单

- [ ] 焦点特征在提示词中出现≥2次？
- [ ] 焦点有明暗/材质对比？
- [ ] 焦点的相对尺度已说明？
- [ ] 焦点的精确位置已说明？
