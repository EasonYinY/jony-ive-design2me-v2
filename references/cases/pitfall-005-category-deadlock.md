# Pitfall 005: 品类锚定死结 (Category Deadlock)

> **版本**: 1.0
> **日期**: 2026-06-21
> **触发**: 2轮迭代，6张图全部失败，用户评分1/10
> **性质**: 泛用性Bug，影响所有消费电子产品AI图像生成任务
> **状态**: ACTIVE — 已修复（见方法14）

---

## 现象

使用 jony-ive-design2me-v2 技能生成"次世代未来感鼠标"时，2轮迭代全部失败：

| 轮次 | 策略 | 提示词长度 | 结果 | 用户评分 |
|------|------|-----------|------|----------|
| Round 1 | 复杂提示词（语义完整优先） | 120-140 words | 像普通鼠标，无识别性 | 1/10 |
| Round 2 | 简化提示词（删除品类词） | 30-40 words | 不像鼠标（像玻璃板/塔/鹅卵石） | 1/10 |

**核心矛盾**：无论怎么调提示词，都无法同时满足"品类识别"和"形态突破"。

---

## 根因：品类锚定死结

```
说"mouse" → 模型渲染默认鼠标原型（无突破）
不说"mouse" → 模型不认识品类（不像鼠标）
说"mouse" + 突破约束 → 模型忽略突破约束（仍是默认原型）
```

### 为什么发生

1. **模型训练偏差**：品类词（mouse/phone/coffee maker）在训练数据中关联极强的默认原型
2. **突破约束弱**："extremely flat""paper-thin"等突破约束在训练数据中关联弱
3. **注意力机制**：模型注意力优先分配给高频强关联词（品类词），忽略低频弱关联词（突破约束）

### 为什么简化提示词也失败

删除品类词后，模型失去"产品认知框架"，只能按"抽象物体"渲染：
- "glass slab" → 像玻璃板/手机/平板
- "tower" → 像音箱/装饰品/加湿器
- "pebble" → 像石头/肥皂/摆件

---

## 修复：品类锚定法（方法14）

### 原理

保留品类词（让模型知道"这是什么"），但立即否定默认原型（让模型知道"不要长这样"），然后给出极端突破形态（让模型知道"要长这样"）。

### 三段式结构

```
1. 品类锚定: "A [品类] that doesn't look like a [品类]"
2. 突破形态: "[极端形态词], [非传统材料], [识别性特征]"
3. 背景控制: "Floats. Pure white. No ground."
```

### 示例

**鼠标**：
```
"A mouse that doesn't look like any mouse you've seen.
Paper-thin glass slab, floating, no buttons visible.
Floats. Pure white background."
```

**咖啡机**：
```
"A coffee maker that doesn't look like any coffee maker.
Black stone slab, recessed cup well, brass drip nozzle.
Floats. Pure white background."
```

**耳机**：
```
"Headphones that don't look like any headphones.
Two carbon rings, kevlar tension lines, no headband.
Floats. Pure white background."
```

### 规则

1. **必须保留品类词**（mouse/phone/coffee maker/headphones）
2. **必须立即否定默认原型**（"doesn't look like" / "unlike any"）
3. **突破形态必须极端**（"paper-thin" "tower-like" "pebble-shaped" "ring-shaped"）
4. **形态描述不得超过10个词**
5. **不得使用数字**（mm/°/ratio）
6. **不得描述交互/功能**

- [ ] 包含品类词？
- [ ] 包含否定默认原型？
- [ ] 突破形态极端？
- [ ] 无数字？
- [ ] 无交互？
- [ ] 无功能？
- [ ] 背景纯净？

## 相关失败模式

| 模式 | 触发 | 现象 | 修复 |
|------|------|------|------|
| 概念覆盖 | 品类词 + 无否定 | 渲染默认原型 | 品类锚定法 |
| 品类丢失 | 删除品类词 | 生成抽象物体 | 品类锚定法 |
| 数字忽略 | 数字（mm/°） | 数字被忽略 | 删除数字 |
| 材料简化 | 复杂材料 | 渲染为塑料 | 单一主材 |
| 背景污染 | 弱背景描述 | 出现环境元素 | 强烈否定 |
| 细节抹除 | 细节>3个 | 所有细节被忽略 | 保留1-2个 |
| 交互不可见 | 交互描述 | 无操作界面 | 删除交互 |
| 文化概念丢失 | 文化杂交 | 概念被忽略 | 删除文化概念 |
| 人体工学不可见 | 人体工学 | 被忽略 | 删除人体工学 |
| 功能说明丢失 | 功能说明 | 不可见 | 删除功能 |

## 流程修复

### 新增 STEP-14.5: 模型能力验证

在STEP-14（审美质量门）和STEP-15（生成提示词）之间新增：

```
STEP-14.5: 模型能力验证
- 输入: 三个方向的DesignIR
- 输出: 验证后的提示词草稿
- 验证清单:
  1. 品类词是否保留？（否→添加）
  2. 突破形态是否极端？（否→强化）
  3. 提示词是否<30 words？（否→压缩）
  4. 是否包含不可渲染字段？（是→删除）
  5. 是否包含数字？（是→删除）
  6. 是否包含交互/功能？（是→删除）
- A/B测试: 生成1张测试图
- 通过→进入STEP-15
- 失败→退回STEP-10重新推导形态
```

### 修订 STEP-15: 提示词输出要求

| 维度 | 历史版（修复前） | 旧版（修复后） |
|------|---------------|---------------|
| 词数限制 | 无限制，语义完整优先 | 30-50 words |
| 品类词 | 删除或保留 | **保留+否定默认原型** |
| 数字 | 强制保留≥2个 | **删除所有** |
| 交互描述 | 强制保留 | **删除所有** |
| 功能说明 | 可保留 | **删除所有** |
| 文化概念 | 自然嵌入 | **删除概念，保留视觉** |
| 方法 | 方法1-12 | **方法13（简化）+方法14（品类锚定）** |
