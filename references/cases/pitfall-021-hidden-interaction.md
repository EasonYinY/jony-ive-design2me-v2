# Pitfall 021 - 交互节点过度隐藏

> **版本**: 1.0
> **日期**: 2026-06-21
> **触发**: 用户反馈"方案没有按键交互的显性特征"
> **严重级别**: MAJOR

---

## 触发条件

1. STEP-11 交互设计时，为追求「极简」过度隐藏交互节点
2. 使用「无按钮设计」等模糊描述
3. 未设计「隐性显性」特征（微凹点/纹理变化/形态暗示/动态反馈）
4. 操作后无可见/可触/可听反馈

## 失败表现

- 图片中找不到任何交互节点
- 用户无法凭直觉知道「这里可以操作」
- 操作后无反馈，用户不确定是否生效
- 产品像「雕塑」而非「工具」
- 交互体验差，学习成本高

## 根因分析

1. **极简过度**: 将「极简」误解为「无交互」，而非「精简交互」
2. **隐性显性缺失**: 未设计微凹点/纹理变化/形态暗示等隐性显性特征
3. **反馈缺失**: 未设计触觉/视觉/听觉反馈
4. **MTH-006 未落实**: 「触觉节点承载确认与信任」未执行

## 修复方法

1. **显性交互特征**: 每个方向必须有 ≥1 个显性交互特征（即使无按钮）
2. **隐性显性设计**: 通过微凹点/纹理/形态/动态反馈实现可发现
3. **反馈设计**: 每个高频操作必须有 ≥1 个可感知反馈
4. **精确描述**: 提示词中必须精确描述交互特征（尺寸、位置、材料、触发方式）

## 验证方式

检查清单：
- [ ] 是否有 ≥1 个显性交互特征？
- [ ] 显性交互特征是否「不突兀但可发现」？
- [ ] 用户能否凭直觉知道「这里可以操作」？
- [ ] 每个高频操作是否有 ≥1 个可感知反馈？
- [ ] 是否避免了「无按钮」等模糊描述？
