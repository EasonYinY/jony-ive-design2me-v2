# Pitfall 013 - 纯审美比例

## 触发条件

1. 比例数字无人体测量或工程约束依据
2. 使用"黄金比例""视觉平衡""和谐感"等纯审美词替代工程依据
3. 尺寸未标注来源（人体测量/功能需求/制造工艺/结构约束）
4. 比例与功能需求脱节（如刀头宽度与胡须覆盖面积不匹配）

## 失败表现

- 比例数字无说服力（如"3:1""5.4:1""6:1"但无法解释为何是这个数字）
- 形态与人体尺度不匹配（如指环 φ20-22mm 但刀头 18mm，手指活动受限）
- 功能区域比例错误（如刀头宽度 18mm 但上唇胡须覆盖宽度 25mm，无法一次覆盖）
- 结构比例失衡（如轨道 R150mm 但底座 200×120mm，视觉重心不稳）

## 根因分析

### 直接原因
STEP-10 形态比例推导时未标注来源：
- 比例数字来自审美直觉而非工程计算
- "黄金比例"被当作通用解，忽视品类特异性

### 深层原因
混淆了"比例精确"与"比例美观"：
- Jony Ive 的比例精确来自每个尺寸有工程或人体依据
- 当前错误做法是用"视觉平衡"等审美词掩盖依据缺失

## 修复方法

### STEP-10 强制字段
每个关键尺寸必须标注来源：

```
尺寸名称 = 数值 ← 来源类型: 具体依据
```

来源类型（四选一）：
1. **人体测量**：基于人体测量数据（如"男性食指关节 φ18-22mm"）
2. **工程约束**：基于物理/力学约束（如"刀头宽度 ≥ 25mm 以覆盖上唇胡须"）
3. **功能需求**：基于功能实现需求（如"电池容量 800mAh 需要 φ12mm×30mm 空间"）
4. **制造工艺**：基于制造限制（如"CNC 铣切最小壁厚 1.5mm"）

示例（正确）：
```
刀头宽度 = 25mm ← 人体测量: 上唇胡须覆盖宽度 20-30mm，取中值
握持直径 = 20mm ← 人体测量: 男性食指关节 φ18-22mm，取中值
机身长度 = 120mm ← 功能需求: 容纳马达(φ18×35mm) + 电池(φ12×30mm) + 刀头(25mm)
壁厚 = 1.5mm ← 制造工艺: CNC 铣切铝合金最小壁厚
```

### 禁止事项
- ❌ "黄金比例 1:1.618"（无品类依据）
- ❌ "视觉平衡"（无具体尺寸）
- ❌ "和谐感"（无工程依据）
- ✅ "刀头宽度 25mm ← 上唇胡须覆盖宽度"（人体测量依据）

### 验证问题
每个比例数字必须能回答：
- 这个数字从哪来？（人体测量/工程约束/功能需求/制造工艺）
- 如果改变这个数字，功能会受什么影响？
- 这个数字在品类内是否有先例？（如现有剃须刀刀头宽度 25-40mm）

## 验证方式

1. 检查每个关键尺寸是否有来源标注（人体测量/工程约束/功能需求/制造工艺）
2. 检查是否存在纯审美比例（黄金比例/视觉平衡/和谐感）
3. 检查尺寸是否与功能需求匹配（如刀头宽度 ≥ 胡须覆盖宽度）
4. 检查尺寸是否与人体测量数据匹配（如握持直径 ≈ 手指关节宽度）
