# Pitfall 015 - 功能未验证

## 触发条件

1. 形态推导后未验证是否能完成核心功能
2. 关键构件（清洁/更换/维修）的可达性未验证
3. 静息状态（存放/充电/维护）未验证合理性
4. 人体工学未验证（握持/佩戴/操作姿势是否合理）

## 失败表现

- 形态无法完成核心功能（如指环式剃须器：18mm刀头在指环内腔无法集成马达和电池）
- 关键区域无法覆盖（如掌托式无法处理鼻翼沟、眼角等细节区域）
- 清洁/更换路径不存在（如刀头模块不可拆卸，指环本体不可浸泡）
- 存放/充电不合理（如轨道式仅限居家镜前，无法旅行携带）
- 人体工学灾难（如手指活动受限，面部自重压力不均，颈部处理困难）

## 根因分析

### 直接原因
STEP-06生成候选后未进行功能验证：
- 候选方向仅基于审美和关系创新，未验证工程可行性
- 人体工学验证缺失（ergonomics未作为独立步骤）

### 深层原因
流程缺少"功能验证"强制检查点：
- 候选生成后直接进入约束检查（STEP-07），跳过功能验证
- 人体工学被隐含在形态推导中，未显式验证

## 修复方法

### STEP-06新增功能验证
每个候选必须回答三个问题：

**Q1: 该形态能完成核心功能吗？**
- 刀头能否接触到所有需要剃的区域？
- 刀头运动方式（往复/旋转）是否适合该形态？
- 动力传输路径（马达→刀头）是否存在？

**Q2: 关键构件的可及性（清洁/更换/维修）如何？**
- 刀头是否可拆卸更换？
- 清洁路径是否明确（水洗/刷洗/自清洁）？
- 维修路径是否存在（螺丝/卡扣/磁吸）？

**Q3: 静息状态（存放/充电/维护）合理吗？**
- 存放是否稳定（不会倾倒/滚动）？
- 充电方式是否便利（无线充电/触点/USB-C）？
- 维护周期和成本是否可接受？

### 否决条件
任一问题答案为"否"或"不确定"，否决该候选。

### 人体工学验证（hand-tool/8小时使用产品）
额外验证：
- 握持/佩戴姿势是否可持续8小时？
- 关键接触点（掌托/指槽/按键）是否有压力分布优化？
- 操作动作是否符合人体自然运动轨迹？

## 验证方式

1. 检查每个候选是否有功能验证三问的回答
2. 检查人体工学验证是否完成（hand-tool产品）
3. 检查否决条件是否严格执行
4. 检查通过验证的候选是否进入下一步约束检查
