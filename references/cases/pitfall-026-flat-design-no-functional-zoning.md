# Pitfall 026: 复杂产品被压扁成 1 块（功能分区缺失）

> **版本**: 1.0
> **日期**: 2026-06-23
> **触发**: 2026-06-23 高逼格咖啡机 6 张图评审，用户原话"中规中矩，很普通"+"功能比较复杂的产品需要做功能的分区，有多个功能区来组成"。vision_analyze 验证 D1/D2/D3 全部被识别为"压扁成 1 块塑料壳"。
> **严重级别**: BLOCKER for 艾维级评分（≤ 5/10 时不可发布）

## 现象

complex product（功能数 ≥ 5）的 3 个方向，全部被生成/渲染为 1-2 个整体造型块：
- 没有"内部功能区"暴露（半透明 / 外露 / 可拆）
- 没有"区-区接缝信号"（chamfer / parting line / material switch / light strip）
- 没有"区-区尺寸突变比 ≥ 3×"（视觉张力完全来自轮廓，缺少内部对比）

**视觉表现**：图像看起来像"1 个塑料/金属/玻璃壳子+1 个底座"，用户反馈"很普通"+"中规中矩"。

## 触发条件

1. brief 是 complex product（功能数 ≥ 5）— 咖啡机/电饭煲/吹风机/无人机/电钻/缝纫机/汽车仪表盘/医疗设备
2. STEP-06 候选生成时，直接以"1 个基础体块 = 1 个方向"为单位
3. STEP-10 形态推导时，整个产品用 1 套尺寸（长×宽×高）描述
4. STEP-15 提示词用单一段落描述（不是分段功能区描述）

## 失败表现

| 维度 | 表现 | 失败码 |
|---|---|---|
| 功能区表达 | 1-2 段（应该 ≥ 4 段） | `aesthetic_ratio_only` |
| 内部结构 | 全部封闭（应该 ≥ 1 区可见） | `structural_misread` |
| 接缝秩序 | 0-1 处接缝（应该 ≥ 3 处） | `material_without_duty` |
| 视觉张力 | 仅外部轮廓（应该来自区-区对比） | `aesthetic_ratio_only` |

## 根因分析

1. **STEP-06 候选单位错位**：把"1 个基础体块"当成"1 个方向"，应该把"1 种功能区排布方案"当成"1 个方向"
2. **STEP-10 推导层级错位**：从"1 个体块"出发 → 应该从"5 个功能区"出发
3. **STEP-15 提示词结构错位**：单一段落 → 应该"分段功能区描述 + 区间接缝"
4. **vision 评审打分偏松**：评审时给"按 DesignIR 一致性"打分（7-8/10）→ 应该"按行业对照（艾维级）"打分（5-6/10 才真实）

## 修复方法（Round 2+ 强制协议）

### 流程层 patch

1. **STEP-06.0 [NEW] 功能区拆解**：brief 落地后立即列 4-7 个功能区，每个写明位置+边界+接口
2. **STEP-06.1 [改] 候选生成**：5 个候选 = 5 种**功能区排布方案**（不是 5 种基础体块）
3. **STEP-06.2 [NEW] 基础体块分配**：选定的功能区排布下，每个功能区独立选基础体块
4. **STEP-10.0 [改] 形态推导**：从 1 套整体尺寸 → 5 区各自的尺寸 + 区间接缝
5. **STEP-10.5 [NEW] 区间接缝信号**：每两区之间明确 1 种接缝类型
6. **STEP-13 [改] DesignIR 新增字段**：
   - `functional_zones`（≥ 3 个）
   - `interior_visibility`（哪些区可见）
   - `inter_zone_seams`（接缝类型列表）
7. **STEP-14 [改] 质量门新增第 12 项**：功能区表达检查（参见 REF-GUIDE-013）
8. **STEP-15 [改] 提示词模板**：单一段落 → 分段功能区描述（5 段 + 接缝段 + 负面约束段 + 背景段）

### 视觉评审严格化 patch

**原评分逻辑**："按 DesignIR 一致性" → 给出 7-8/10（偏松）

**新评分逻辑**（强制）："按艾维级行业对照" → 给出真实分数
- ≤ 5/10：明显压扁/品类俗套 → BLOCKER，退回 STEP-10
- 5-6/10：有分区但不显著 → 退回 STEP-10.5
- 6-7/10：基本达到艾维级 → 可发布但需要小修
- 8-9/10：达到 LoveFrom 水平 → 可发布
- 9-10/10：超越（极少） → 可发布 + 标记为标杆

**自检问题**（vision_analyze 必跑）：
- Q1: 设计是否表达了 ≥ 4 个功能区（视觉上可识别）？
- Q2: 是否 ≥ 1 个功能区半透明/外露/可拆？
- Q3: 是否 ≥ 1 对功能区之间有 ≥ 3× 尺寸突变？
- Q4: 每个功能区是否有独立材料或表面处理？
- Q5: 与 iMac G4 / Linn LP12 / Beosound 1 对照，张力从何而来？

任一为"否" → 触发本 pitfall。

## 验证方式

每张图生成后，**双评分**：
1. DesignIR 一致性分（"是否表达了 DesignIR"）— 原评分
2. 艾维级行业对照分（"是否达到 LoveFrom 水平"）— 新评分

两分差距 ≥ 1.5 分 → 提示词有"形式合规但行业平庸"问题，需要重写
