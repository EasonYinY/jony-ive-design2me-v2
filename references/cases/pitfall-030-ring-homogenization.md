# Pitfall 030 - 环形结构同质化（Ring Homogenization）

> **日期**: 2026-06-24
> **触发**: 用户反馈"为什么总会输出上环加下环的这种设计，每一次都会有？"
> **严重级别**: 🔴 BLOCKER for 含环 brief
> **强制范围**: STEP-06 候选生成 + STEP-08 反俗套检查

---

## 1. 问题定义

当 brief 允许或暗示"环"元素时（如飞行器、灯具、穿戴设备），设计推导会**系统性默认**为"上下双环"或"同心环"结构，导致：
- 三个方向本质同质化（都是环的变体）
- 视觉输出千篇一律（上环+下环+连接件）
- 用户审美疲劳（"每一次都会有"）

---

## 2. 根因分析

### 2.1 拓扑几何体选择惯性

| 惯性路径 | 错误示例 | 后果 |
|----------|----------|------|
| "环 = 稳定" | 上环升力+下环稳定 | 上下环成为默认公式 |
| "环 = 包围" | 环包围舱体/人 | 同心环成为默认公式 |
| "环 = 旋转" | 环作为旋翼外框 | 单环+叶片成为默认公式 |

### 2.2 反形态预设检查失效

STEP-05 反形态预设检查**只检查已知产品形态**（直升机/飞机/汽车），但**不检查抽象形态同质化**（上下环/同心环/单环阵列）。

### 2.3 六轴差异门漏洞

STEP-09 六轴差异门检查材料/工艺/颜色/形态/关系/结构差异，但**如果三个方向都含环，形态差异只是"环的大小/数量不同"，不是本质不同**。

---

## 3. 强制规则（Ring Diversity Mandate）

### 3.1 环位置多样化（必须执行）

若 brief 含环元素，三个方向的环必须分布在**三个不同位置**：

| 位置 | 英文 | 功能隐喻 | 示例 |
|------|------|----------|------|
| **顶部** | top / overhead | 伞/庇护/覆盖 | 顶部单环如伞盖 |
| **底部** | bottom / undercarriage | 基座/支撑/起落 | 底部环状起落架 |
| **背部** | rear / back | 推进/稳定/尾翼 | 背部双环如昆虫复眼 |
| **中部** | mid / waist | 束缚/分割/连接 | 环绕腰带如腕表 |
| **前部** | front / nose | 引导/进气/撞击 | 前部环状进气口 |
| **四周** | surround / peripheral | 包围/阵列/分布式 | 多环蜂窝围绕中央 |

**硬约束**: 三个方向的环位置必须**两两不同**（如顶部/背部/中部），不允许任何两个方向共享同一位置。

### 3.2 环数量多样化（必须执行）

| 数量 | 适用场景 | 禁止 |
|------|----------|------|
| **单环** | 顶部庇护/底部起落/中部束缚 | 单环+同心单环（仍是同心环） |
| **双环** | 背部推进/上下稳定 | 双环必须**非同心**（如水平并列/错位） |
| **多环** | 四周阵列/蜂窝分布式 | 多环必须**非同心阵列**（如蜂窝/随机） |
| **无环** | 至少一个方向完全无环 | 三个方向都含环 = 同质化风险 |

**硬约束**: 三个方向的环数量组合必须**不完全相同**（如单环/双环/无环，或单环/双环/多环）。

### 3.3 环关系多样化（必须执行）

环与主体（舱体/人/功能核心）的关系必须不同：

| 关系 | 英文 | 隐喻 | 示例 |
|------|------|------|------|
| **悬挂** | suspended | 伞-人 | 环在上，主体悬挂于下 |
| **支撑** | supported | 底座-建筑 | 环在下，主体承托于上 |
| **推进** | propulsive | 背-翼 | 环在后，推动主体前进 |
| **束缚** | bound | 腰带-身体 | 环在中，环绕主体分割 |
| **包围** | surrounded | 蜂窝-核心 | 环在四周，主体在中央 |
| **引导** | leading | 鼻-气流 | 环在前，引导主体方向 |

**硬约束**: 三个方向的环-主体关系必须**两两不同**。

---

## 4. 检查清单（STEP-06 生成后必跑）

```
### 环多样性检查
- [ ] 三个方向的环位置是否两两不同？（顶部/底部/背部/中部/前部/四周）
- [ ] 三个方向的环数量组合是否不完全相同？（单环/双环/多环/无环）
- [ ] 三个方向的环-主体关系是否两两不同？（悬挂/支撑/推进/束缚/包围/引导）
- [ ] 是否有至少一个方向完全无环？（强制）
- [ ] 是否所有环都是同心环？（禁止，必须否决）
- [ ] 是否所有环都是上下堆叠？（禁止，必须否决）
```

---

## 5. 失败码

| 失败码 | 描述 | 严重性 |
|--------|------|--------|
| `ring_homogenization.position_same` | 两个方向环位置相同 | 🔴 BLOCKER |
| `ring_homogenization.quantity_same` | 三个方向环数量完全相同 | 🔴 BLOCKER |
| `ring_homogenization.relationship_same` | 两个方向环-主体关系相同 | 🔴 BLOCKER |
| `ring_homogenization.all_have_ring` | 三个方向都含环（无无环方向） | 🟠 HIGH |
| `ring_homogenization.concentric` | 存在同心环结构 | 🔴 BLOCKER |
| `ring_homogenization.vertical_stack` | 存在上下堆叠环 | 🔴 BLOCKER |

---

## 6. 修复方法

若触发失败码，按以下优先级替换：

1. **保留一个方向为"无环"**（强制）
2. **将同心环改为偏心环/错位环/倾斜环**
3. **将上下堆叠改为水平并列/前后错位/环绕包围**
4. **将单环功能从"承力"改为"庇护/推进/束缚/引导"**
