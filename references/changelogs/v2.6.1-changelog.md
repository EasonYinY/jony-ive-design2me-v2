## 决策记录

1. **决策 1:KB 不进 graph,而通过"路径存在性"在 references/README.md 索引表维护**
   - 备选方案 A:让 KB README 进 graph(需修改 validate_reference_graph.py 的 fail-fast 行为)
   - 选择理由:fail-fast 在其他 cases/iterations 文件处会抛错,改动跨出范围
   - 实际效果:KB 整体跳过 graph 加载,但 [references/README.md](../README.md) 的 `REF-KB-INDEX` 行通过字符串匹配验证路径存在性,仍能保证"链接路径存在"

2. **决策 2:运行 reference 不嵌入 KB 原文链接到核心方法结论**
   - ive-thinking-models.md 的方法论主体保持原样,只在 frontmatter 之后加"上游 KB 反向链接"段
   - 不破坏 running reference 的稳定快照承诺

3. **决策 3:STEP-15 的 KB 链接加了"严禁直接复制"的强约束**
   - KB 的图像提示词库/实战模板/Prompt 协议等是**参考语料**,不是 PromptBundle 内容
   - 防止误用破坏 prompt-contract 字符边界(90-160 字符范围)
