## 审计报告三大指令

### 指令一：空间布局与尺度前置硬验证

**问题诊断**: STEP-04 人体测量数据（ISO 7250）在 STEP-10 形态推导时未作为硬性数学边界约束，导致 2.2m×1.8m×1.0m 飞行器尺寸工程失效（单人半躺姿纵向投影已达 1.4-1.6m）。

**修复措施**:
- 新增 **STEP-04.5 空间包络硬验证（Spatial Package Validation Matrix）**
- 强制公式：`[净尺寸需求 + 结构厚度容差 + 安全防护边界 = 最终输出数值]`
- 载具类公差 ±10%，电子设备 ±0.5mm
- 未通过校验标注 `UNVALIDATED` 并回退重新推导
- 详见 `references/cases/pitfall-055-scale-fact-disconnect.md`

### 指令二：视觉等效洗涤算法

**问题诊断**: STEP-12 材料职责矩阵中的工程账本数据（寿命 20 年、热压罐成型、超声波检测）直接注入提示词，对 Nano Banana Pro 造成语义噪音（"20年"被错误具象化为陈旧纹理）。

**修复措施**:
- 新增 **方法17：视觉等效洗涤（Visual Equivalent Wash）**
- 新增 **方法18：尺度缩放滤网（Scale Zoom Filter）**
- `compile_prompt.py` 新增 `VISUAL_EQUIVALENT_MATRIX`（10 种材料映射）
- `compile_prompt.py` 新增 `ENGINEERING_NOISE_WORDS` 自动过滤（20+ 词汇）
- 工程账本数据 → 光学物理表征（IOR/Roughness/Anisotropy/Micro-texture）
- 详见 `references/guides/prompt-engineering.md` 方法17/18

### 指令三：五段式物理空间协议深度定制

**问题诊断**: Block 2 材质职责描述导致模型材质贴图化；Block 3 精确微观尺寸（0.5mm）与宏观体量（2m）语义冲突；Block 4 "Floats" 产生水面漂浮次生幻觉。

**修复措施**:
- **Block 2**: 材质职责 → 核心拓扑与交接关系（Visualized Intersect），强制物理交接动作词
- **Block 3**: 精确微观尺寸 → 微观物理交接线（hairline shadow line / crisp alignment split）
- **Block 4**: Floats → 消失设计与行为触点（Light interaction defines the forms）
- **Block 5**: 新增 `no cybernetic patterns, no sci-fi mechanical ribbing, no background props`
- 详见 `references/process/end-to-end-workflow.md` V3.0 审计增强
