## 决策记录

1. **决策 1:用 path-based 比较替代 basename**
   audit 之前用 `by_basename.get(target)` 查找,ledger 改成相对路径后改为遍历查找
   
2. **决策 2:KB 原文 28 篇不参与运行 reference 闭环**
   0 改写原则优先 + fail-fast 容错 + KB/README.md 例外(参与闭环)
   
3. **决策 3:humanizer 严格限制为 3 个真正的 AI 套话**
   234 次"候选"实际是领域术语,替换会破坏专业语义
   极致识别性(评分维度)、触达(人体工学)、对齐(机械精度)都是术语
   
4. **决策 4:不重写结构、不润色 frontmatter**
   frontmatter 是 graph 验证的依据,改 frontmatter 会破坏引用闭环
   
5. **决策 5:不润色 ASCII 架构图、代码块、表格、链接**
   这些是技术文档的硬契约,润色它们会产生歧义
