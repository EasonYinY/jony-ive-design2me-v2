## 定位澄清

`references/knowledge-base/` 是**上游参考/输入**目录,**不是** jony-ive-design2me-v2 的运行知识。运行知识仍由技能内置的 `references/core/`、`references/design/`、`references/direction/` 等稳定快照承担(自包含承诺)。

| 类型 | 来源 |
|---|---|
| 运行知识 | `references/core/`, `references/design/`, `references/direction/`, `references/prompt/`, `references/quality/`, `references/workflows/` |
| 上游参考 | `references/knowledge-base/`(本目录) |
| 元数据 | `references/README.md`, `references/known-gaps.md`, `references/source-index.md`, `SOURCE_LEDGER.csv` |

## 决策记录

1. **决策 1:不做精简合并**
   - 备选方案 A:把 28 篇合并为 3-5 篇运行摘要
   - 用户原话:"不可以有任何 token 焦虑,不会因为 token 导致的文字描述优化,减少,删减,简化的描述"
   - **选择**:字节级完整镜像,不合并、不精简、不删字

2. **决策 2:不修改运行知识**
   - `references/core/ive-thinking-models.md` 等是稳定运行快照,不应被外部知识蒸馏库污染
   - **选择**:在 SKILL.md 加 1-line 路由块,运行知识文件 0 改动

3. **决策 3:不注入 frontmatter 到原始文件**
   - 镜像文件保持原资料库格式(部分有 `> **版本**:x.x` / `> **日期**` 注释,部分没有)
   - **选择**:不修改原始文件,在 KB README 中集中登记元数据

4. **决策 4:版本号小幅提升**
   - v2.5.0 → v2.6.0(新增目录 + 新增索引行,符合 semver)
