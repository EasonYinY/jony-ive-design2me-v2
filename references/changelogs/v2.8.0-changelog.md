# v2.8.0 — 版本号污染清理 + 唯一权威源协议

> 用户原话（2026-06-23）："清除所有乱七八糟的版本号，统一一个版本，仅在版本记录的文件和 skill.md。并严格遵守以后升级仅修改这两个地方的版本号，一个是版本号，一个是版本记录"

## 变更内容

### 协议层（永久有效）

**版本号唯一权威源协议**：
- 版本号**只允许**出现在 2 个地方：
  1. `SKILL.md` frontmatter 的 `version:` 字段
  2. `references/changelogs/v{X.Y.Z}-changelog.md` 文件名 + changelog 内部 frontmatter
- **其他任何文件**（references/ 60+ .md、scripts/、workflows/、cases/、guides/、core/、design/、prompt/、quality/、README、handoff-*）**任何位置**（正文、frontmatter、内联引用、文件链接、注释、表格内）**一律不得出现版本号**
- 备份目录 `*-BACKUP-v*` 的目录名是历史 artifact，不动；备份目录里的文件按其原版本保留

### 升级纪律

以后任何升级（含本版本）：
- ① 改 `SKILL.md` frontmatter `version:` 字段（唯一一处 frontmatter 升级）
- ② 新建 `references/changelogs/v{X.Y.Z}-changelog.md`（唯一一处 changelog 升级）
- ③ **不得**改任何其他文件的版本号标记
- ④ **不得**在其他文件正文中加"v{X.Y.Z} 新增""v{X.Y.Z}+"等标记
- ⑤ 历史内容里的版本号引用（如 "v1.1 升级""v2.0.0+"）保留原样**作为历史记录**，但**新写入内容**禁止使用

### 新增验证

- `scripts/check_version_pollution.py`（NEW）— 全文扫描验证脚本：检测 SKILL.md frontmatter + changelog 文件名/changelog 内部 frontmatter 之外的所有版本号污染，输出按文件 + 行号 + 模式分类的违规清单，CI 友好退出码
